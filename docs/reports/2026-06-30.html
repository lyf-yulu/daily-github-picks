awesome-gamedev-agent-skills 是一个面向 AI 编程代理（Agent）的游戏开发技能集合，提供 66 个原创、版本锁定的技能文件以及一个智能路由（Router），用于将游戏开发专业能力注入到 Claude Code、Cursor、Codex、Copilot、Gemini CLI、Windsurf、Kiro 等主流 AI 编程工具中。项目采用便携的 SKILL.md 格式，单一文件可跨多种代理工具原生加载，无需任何转换。核心机制是按需加载：开发者只需用自然语言描述需求（例如“给我的 Godot 角色添加二段跳”），Router 会自动检测项目所属引擎（通过 project.godot、*.uproject 等指纹文件）、解析任务意图，然后精准加载对应的引擎基础技能、概念技能和题材技能，避免上下文污染。覆盖引擎包括 Godot 4.x、Unity 6 LTS、Unreal、Phaser、PixiJS、three.js、Bevy、pygame、LÖVE 与 Roblox，并扩展到平台运动、Tilemap、物理、动画、Shader、多人联机、存档系统、程序化生成、Roguelike、RPG、Steam/itch 发布等横向主题。所有技能均基于一手官方文档撰写、与具体引擎版本绑定，并通过 validate-skills.py 校验脚本保证一致性。安装方式简洁，一条 npx skills add 命令即可完成所有代理工具的自动适配，也提供 Claude Code 插件市场作为替代入口。
它把“给 AI 代理装游戏开发大脑”这件事产品化了：用一个 Router 解决了多技能选择难题，用版本锁定 + 一手文档解决了 LLM 在游戏引擎 API 上常见的幻觉，又用 SKILL.md 标准格式打通了 Claude Code、Cursor、Copilot 等异构生态。对于希望让 AI 真正胜任 Godot、Unity、Unreal 项目协作的开发者，这是一个可直接复用的工程级范式，也为“Agent Skills”这一新兴生态提供了高质量的开源参考实现。

Jobsmith 是一款针对台湾求职市场的开源多代理（multi-agent）AI 求职 co-pilot，覆盖从找职缺、履历健检、客制投递包生成到模拟面试的完整流程。其核心差异化在于默认对接本机已登入的 Claude Code 或 Codex CLI 订阅作为 AI 引擎，免去自行申请 API key 的门槛，也支持 Ollama、llama.cpp 等本机模型以及任何 OpenAI 相容端点的 BYOK 模式，强调资料本机化与隐私可控。功能上，它会上传履历后由 AI 推导关键词，并行抓取 104、Yourator、LinkedIn、Cake 等台湾主流职缺来源，并以 SSE 分批串流方式即时评分排序；用户可在职缺工作台一键触发投递包生成，由多个 agent 协同完成 JD 解析、匹配评分、公司情报、客制履历、求职信、面试准备与品管反思。整个生成过程作为背景工作运行，离开页面或重新整理都不中断，多个职缺可并行处理，完成后进入「我的投递包」按生命周期状态（进行中→待审→已核可）逐一核可、汇出 Word 或开启模拟面试。技术层面基于 FastAPI 后端 + LangGraph StateGraph 多代理编排（各自记忆体 checkpointer 支持并行），前端使用 React 19 + Vite + TypeScript，并以 PyInstaller 打包成 Windows exe 与 macOS unsigned dmg。CLI 后端还做了模型自动分层：解析用 haiku、生成用 sonnet、Critic/Supervisor 用 opus。
在 AI Agent 项目同质化严重的当下，Jobsmith 提供了一个少见的『订阅复用』思路——把开发者本机已付费的 Claude Code 或 Codex CLI 当成免 key 推理引擎，并辅以本机模型与 BYOK 三档可切换的后端，大幅降低了 multi-agent 应用的使用门槛。配合 LangGraph 背景并行编排、Supervisor 反思迴圈带来的可量化品质提升，以及对台湾求职市场（104 / Cake / Yourator）的高度在地化整合，使它既是实用的求职工具，也是研究多代理工程实践与本地化 LLM 落地的优质参考。

Error Discovery Skill 是一个面向AI智能体的错误分析技能包，核心是一份 SKILL.md 指令文件，任何能读取文件的智能体（如 Claude Code）都可遵循其规则执行。它将 JSONL/CSV/JSON 格式的 LLM 输出或追踪数据作为输入，自动识别内容类型（文章、智能体轨迹、代码、结构化输出等），并基于格式塔原则设计可视化编码——用颜色区分类别、用间距体现层级、用透明度表达重要性。随后，它使用 Python 标准库服务器生成一个零依赖的单文件 HTML 审阅应用，对数据进行聚类并挑选多样化的初始样本（聚类代表加随机抽样）。在交互循环中，技能持续监控人工标注，将自由文本笔记归类为失败模式，跟踪覆盖度，并主动提出新样本以填补盲区。整体定位是把人类评审与智能体自动归纳结合，加速 LLM 应用的错误分析与故障模式发现。
