为编码 Agent 打造的项目持久记忆层

Node.jsMarkdownShellGit HooksMCP
brain.md 是一个面向编码 Agent（如 Claude Code、Codex 等）的持久化记忆层标准与工具集。它通过在项目仓库中维护一份纯 Markdown 的 BRAIN.md 协议文件以及 brain/ 目录，将项目中难以从代码和 git 历史还原的耐久知识——决策理由、需求约束、放弃的方案——固化下来，使其跨越会话、机器和模型继续生效。项目本身是工具包而非具体大脑：通过 brain-setup 脚手架、brain-bootstrap 知识种子、brain-page 读写手册和 brain-ingest 摄取等 Skills 协作。其核心是一个零依赖的 Node CLI，所有写入都经由 CLI 完成，页面包含可重写的 compiled_truth 与只追加的 timeline，update-truth 通过原子写入保证理解变更必留痕迹。设计上以 Agent 无关、仓库原生、构造即正确为原则，无需校验器即可保证不变量。

opencodex ⭐ 111

让 Codex 接入任意 LLM 的本地代理

Bun 1.1+TypeScript/JavaScriptNode.js (npm 分发)OpenAI Responses APIAnthropic Messages APIGoogle Gemini APIAzure OpenAIOAuth 2.0launchd / systemd / Task Scheduler
opencodex 是一款轻量级本地代理工具，定位为 OpenAI Codex 的「通用提供商代理（Universal Provider Proxy）」。它将 Codex 的 Responses API 翻译为各家 LLM 提供商的协议，使 Codex CLI、App 与 SDK 无需等待 OpenAI 官方支持，即可直接调用 Claude、Gemini、Grok、GLM、DeepSeek、Kimi、Qwen、Ollama 等 40 多个模型服务。项目通过 5 类协议适配器覆盖 Anthropic Messages、Google Gemini、Azure OpenAI、OpenAI Responses 透传以及通用 OpenAI 兼容 Chat Completions 端点，完整支持流式输出、工具调用、推理 token 与图像等双向能力。安装方面通过 npm 全局安装 @bitkyc08/opencodex，运行在 Bun 1.1+ 之上，原生支持 macOS（launchd）、Linux（systemd 用户单元）和 Windows（Task Scheduler），无需 WSL。使用上提供 ocx init 交互式配置、ocx start 启动代理、ocx gui 打开本地 Web 仪表盘（localhost:10100）来添加提供商，支持 OAuth 登录（xAI、Anthropic、Kimi）、API Key 与环境变量引用，并能自动从 /v1/models 发现模型。模型路由采用 provider/model 语法，路由后的模型会出现在 Codex App 的选择器中并支持推理强度控制。此外还提供子代理委派、为非 OpenAI 模型注入 gpt-5.4-mini 的联网搜索与图像理解 sidecar、后台服务与 codex-shim 两种自启方式，以及干净卸载恢复原生 Codex 配置等能力。
OpenAI Codex 生态此前严重依赖官方模型支持节奏，opencodex 以一个本地代理巧妙解耦，让开发者能用 Claude、Gemini、Grok、本地 Ollama 等任意模型驱动 Codex 工作流，并保持流式、工具调用、推理 token、图像等高级特性的完整保真度。其在协议翻译、跨平台服务托管、OAuth 认证、模型自动发现、子代理路由以及干净可逆的注入卸载机制上的工程完成度，使其不仅是 hack 式适配，而是一个可日常依赖的生产力工具，对希望摆脱单一供应商绑定、追求多模型混合编排的开发者具有较高参考价值。

UmaDev ⭐ 102

Rust 1.87+Claude Code / Codex / OpenCode CLITUI / CLIOpenAPI 契约BM25 + 向量检索Mermaidnpm 分发
UmaDev 是一个本地运行的 AI 项目总监 Agent，脱胎于早期的 super-dev 项目，使用 Rust 重写为单二进制跨平台工具。它不直接调用模型 API，而是加载用户已登录的 Claude Code、Codex 或 OpenCode 作为大脑，通过常驻持续会话驱动一支由产品经理、架构师、UI/UX 设计师、前后端工程师、QA、安全红队、运维和总监组成的虚拟团队，把一句需求交付成可上线产品。核心设计围绕 UMADEV_HOST_SPEC_V1 规范，建立了智能路由、可视化 DAG 计划、固件注入加 repo-map 代码库感知、9 阶段流水线、每步确定性验收地板和交付证明六层闭环。干活角色串行写主会话，评审角色 fork 只读分叉并行审查，通过黑板文件和结构化裁决沟通，避免多 Agent 互聊放大幻觉。系统内置 112 条治理规则、OpenAPI 契约校验、本地 BM25 加向量混合检索的知识库，跑完后产出 PRD、架构文档、UIUX 文档、HTML 成绩单和打包的 proof-pack 作为可审计证据。强调 fail-open 兜底、持久化对话与跨会话续接、踩坑沉淀自学习，让 AI 编码从聊完即完升级为带证据链的工程交付。
在多数 AI 编码工具还停留在补全和单 Agent 写代码时，UmaDev 用规范驱动的 9 阶段流水线、确定性地板和团队角色席位重新定义了 AI 项目交付，配合 proof-pack 审计证据和不绑定模型端点的设计，给出了一条把 AI 编码工程化、可治理、可验收的可行路径。

Calesthio 自称是全球首个开源的、具备智能体（Agentic）能力的视频制作系统。其核心定位是把现有的 AI 编程助手（例如 Claude Code、Cursor、Copilot 等）扩展为一套完整的视频制作工作室，让用户通过自然语言指令就能完成从脚本、剪辑、特效到导出的全流程视频生产。项目以三组关键数字概括其能力规模：12 条流水线（Pipelines）覆盖视频制作的不同阶段，52 个工具（Tools）提供原子化的操作能力，以及 500+ 个智能体技能（Agent Skills）作为可复用的高层动作单元。这种「Pipelines + Tools + Skills」的三层架构是当前 Agent 工程化的典型范式：流水线负责编排，工具负责执行具体动作（如调用 ffmpeg、剪辑、字幕生成、转码等），技能则封装领域知识与最佳实践，让 LLM 在不同情境下选择合适的执行路径。由于 README 信息有限，具体技术栈与依赖需结合仓库源码进一步确认，但从「agentic」「coding assistant」等关键词推断，项目极可能基于 Python 生态、配合 ffmpeg 等视频处理工具，并以 MCP 或自定义 Agent Skill 协议与编程助手对接。对开源社区而言，它把通用编程 Agent 的能力域从代码扩展到了多媒体创作，是 Agent 应用场景外延的一次有趣尝试。
该项目把 Agent 应用从代码补全推向了多媒体内容生产这一更复杂的工作流，验证了「工具 + 技能」式 Agent 架构在非编码领域的可迁移性；同时它以开源方式提供了一个可参考的大规模 Skills 组织样板（500+ 技能），对希望搭建自有 Agent 体系的团队具有结构借鉴价值。
