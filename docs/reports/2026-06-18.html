bazi-ziwei-skill 是一款遵循 SKILL.md 开放标准的命理分析 Skill，可装入 Claude Code、Claude Desktop、Codex、Cursor、Hermes、OpenClaw 等支持该标准的 AI Agent。项目针对纯 LLM 排盘易出错的痛点，将「排盘」「补层」「分析」「渲染」拆分为四个独立环节：八字四柱、紫微十二宫、大运流年由内置算法库（源自开源项目 Yiqi，MIT 协议）确定性计算，避免 LLM 在日柱、日主、格局等基础环节出错；在排盘之上再补一层格局、旺衰、调候、刑冲合害、盖头截脚等算法，为 LLM 提供有据可依的输入；LLM 只负责文字分析，HTML 视觉则交给固定模板填槽。功能上提供八字独立、紫微独立、八字+紫微综合印证三种分析模式，支持 Markdown 长文深度版与单文件 HTML 水墨风海报版两种呈现形态，海报包含紫微 12 宫盘、八字四柱盘与六维交叉对账，可截图分享。算法层基于 Node.js 18+ 与 TypeScript，仅依赖 lunar-typescript 完成农历换算，所有排盘在本地完成、无需联网，隐私友好。仓库还附带合成示例命盘与海报成品，并提供命令行直接排盘的脚本入口。
在 LLM 命理类工具普遍依赖模型「猜排盘」的现状下，本项目以「算法精准排盘 + 格局补层 + 双体系交叉印证」的工程化思路给出了一个可复用范式：把确定性计算与生成式分析严格分层，并通过 SKILL.md 标准实现跨 Agent 通用。这一拆分思路对其他需要严谨基础数据的垂直领域 Skill（法律、医疗、财税等）具有直接借鉴价值，海报式 HTML 输出也展示了 LLM 结构化生成 + 固定模板渲染的优雅落地方式。

architect-loop 是一个面向 Claude Code 的技能（Skill）项目，将跨厂商的 AI 代理协作模式产品化：由 Claude（代号 Fable）担任架构师负责规划、设计与评审，由 OpenAI 的 GPT-5.5 Codex 担任建造者负责实现与研究，仓库本身则充当唯一的共享记忆。项目通过两条命令暴露能力：/architect 驱动构建循环，Fable 先在 docs/gates/ 写入只读验收门，再把任务切成 1–4 条互不重叠的 lane，每条 lane 在独立 git worktree 中由全新 codex exec 进程并行执行，最后 Fable 亲自跑门控命令、对照 diff 与意图判定再合并；/architect-research 驱动研究循环，先由廉价 Codex 侦察员勘察主题，Fable 据此为每个课题动态设计 3–6 条研究 lane，并行 Codex 研究员在严格预算与引用规范下作业，最后由 Fable 完成对抗性核验并独自撰写报告。设计上强调外部冻结门、worktree 隔离、新会话评审、HANDOFF 短目录式记忆等经过文献佐证的工程实践，默认无需 API Key，直接复用用户已有的 Claude Code 与 ChatGPT 订阅额度。
这是少见的把'跨厂商代理协作'从 Twitter 灵感落到可复现工程的开源实现：每条规则都附带文献依据（DESIGN.md），并直面代理工程的真问题——规划薄弱、共享文件互相踩踏、测试可被博弈、长上下文记忆腐化。对正在搭建多代理系统的团队，它提供了一份可直接借鉴的拓扑与门控范式。

MetaHarness 智能体脚手架工厂 ⭐ 164

MetaHarness（agent-harness-generator）是 ruvnet 开源的「元工具链生成器」，定位是为每个代码仓库铸造一套专属的 AI Agent Harness，而非又一个 Agent 框架。用户只需提供 GitHub URL 或空白模板，即可在浏览器或终端中通过 `npx metaharness` 在 60 秒内生成一份可发布到 npm 的 .zip 包，包含仓库感知的 CLI、本地 MCP 服务器、按项目作用域的记忆命名空间、由实际文件结构推导的技能、治理策略，以及由见证者签名的发布溯源。生成的 harness 默认适配九大宿主：Claude Code、OpenAI Codex、pi.dev、Hermes、OpenClaw、RVM 硬件隔离沙箱、GitHub Copilot、OpenCode 以及非交互式的 GitHub Actions CI/CD。项目还内置 `metaharness score` 静态评分（不执行仓库代码），可在搭建前预估适配度、构建成功率、工具安全性与单次运行成本；并提供 `@metaharness/router` 路由器，根据自有评测数据自动选择性价比最高的模型，DRACO 基准实测可让小模型以约十分之一成本达到前沿模型质量。MCP 被设计为可插拔适配面，默认禁用网络、Shell、写文件，配套 `harness mcp-scan` 工具像 `npm audit` 一样静态审计代理工具的危险授权。仓库已通过 568 项测试，并提供 19 个垂直模板（编码、DevOps 等）开箱即用。
在 Agent 框架同质化严重的当下，MetaHarness 反向定位为「Agent 框架的工厂」，把模型当作可替换部件，把 harness（脚手架本身）作为产品；它将仓库分析、记忆作用域、治理策略、签名发布、跨宿主适配与成本路由整合为一条可复现的流水线，且默认离线、默认拒绝危险权限。这种「先评分再搭建、生成即拥有、按仓库定制」的范式，对希望在企业内规模化推广 AI 编码代理而又担心安全与成本失控的团队，具备直接落地价值。

Omnigent 是一个开源的 AI 智能体框架与元 harness（meta-harness），旨在为 Claude Code、Codex、Cursor、Pi 以及用户自定义的 Agent 提供统一的编排与协作层。开发者无需重写代码即可在不同 Agent harness 之间自由切换或组合，并能在同一会话中让多个 Agent 协同工作，例如让一个 Agent 评审另一个 Agent 的产出，或将任务拆分给各有所长的 Agent 并行处理。项目主打跨设备协作：会话可在终端、浏览器、手机之间无缝延续，消息、子 Agent、终端与文件均保持同步，团队成员还可共享会话进行实时观摩、共驾或 fork 分叉。模型层面兼容官方 API Key、Claude/ChatGPT 订阅及任意兼容网关，并可选 Databricks 工作区作为模型提供方。运行时支持本地 macOS seatbelt、Linux bubblewrap 沙箱隔离，也可推送到 Modal、Daytona、Islo 等云端一次性沙箱中执行，免去本地依赖。治理方面提供策略系统（policies），可在风险操作前暂停等待审批、限制工具调用范围或设定花销上限，粒度覆盖整个服务器、单个 Agent 或单条会话。仓库内置 Polly（多 Agent 编码编排者）与 Debby（双脑头脑风暴伙伴）两个示例 Agent，便于快速上手。整体基于 Python 3.12+，通过 uv/pip/Homebrew 一键安装，附带本地 Web UI 与 macOS 桌面应用。
在 AI Coding Agent 百花齐放、各家 harness 各自为政的当下，Omnigent 把 Claude Code、Codex、Cursor、Pi 等异构 Agent 抽象到同一编排层，让「多 Agent 互审、跨厂商协同」从概念变成可落地的工作流，配合策略治理与云沙箱，明显比单一 harness 更接近企业级生产可用。跨设备会话与团队共享会话的设计，也指向了 Agent 协作工具的下一个形态。
