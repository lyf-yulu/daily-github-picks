BaoCut Agent Skill ⭐ 235

让 AI 代理驱动 BaoCut 视频剪辑

skills.sh Agent Skill 规范Shell 脚本Markdown（SKILL.md 路由）macOS CLI（baocut-cli）Node.js（仅安装器 npx skills）
BaoCut Agent Skill 是一个开源的 Agent Skill 项目，用于让 AI 编程代理（如 Claude Code、Codex 以及任何兼容 skills.sh 的代理）通过自然语言指令来驱动 macOS 应用 BaoCut 的命令行工具。BaoCut 本身是一款专注于视频处理的 Mac 应用，具备转写（transcribe）、字幕生成与翻译（subtitle & translate）、说话人识别审阅、以及针对访谈类视频的智能剪辑（cut talking-head）和导出等能力。本项目并不包含视频处理引擎，而是作为一层轻量的 Skill 封装，将 BaoCut.app 内置的 `baocut` CLI 暴露给 AI 代理，让用户用一句“转写并翻译字幕为中文”即可完成从转写到润色再到翻译的完整链路。项目结构简洁，核心目录 `skills/baocut/` 包含作为代理入口的 `SKILL.md` 路由文件、按任务分类的 `references/` 参考文档，以及一个用于定位 `/Applications/BaoCut.app` 内 CLI 二进制的 `bin/baocut` 解析脚本。安装方式支持通过 `npx skills add` 一键全局部署，也支持不依赖 Node.js 的手工 git clone + symlink 方案，安装后可自动为 Claude Code、Codex 建立软链，实现真正意义上的“自然语言驱动本地 GUI 应用”。
该项目展示了 AI Agent 与本地 GUI 应用融合的一种典型范式：不重造轮子，而是通过 Skill 层把成熟桌面软件的 CLI 能力暴露给大模型代理，从而让 Claude Code、Codex 这类通用编程代理具备专业视频处理能力。对于关注 skills.sh 生态、探索 AI 代理如何调用本地专业工具、或希望为自己 Mac 应用增加“AI 可调用”入口的开发者，具有直接参考价值。

LLMVault ⭐ 200

面向AI安全的OWASP LLM Top 10靶场

Python 3.10+FlaskGunicornDocker / Docker ComposeFernet (cryptography)SVG / PNG 前端渲染
LLMVault 是一个刻意构造漏洞的 CTF 风格训练平台，聚焦 OWASP Top 10 for LLM Applications（2025），可视为面向 AI 安全的 WebGoat / KubeGoat。项目共提供 25 个实验，分为三个逐级解锁的层级：核心层覆盖 OWASP LLM 十大类别（提示注入、敏感信息泄露、供应链、数据与模型投毒、输出处理不当、过度自主、系统提示泄露、向量与嵌入、错误信息、无界消耗）各一个实验；进阶层包含 10 个多轮对话挑战，涵盖角色扮演越狱、二阶注入、代理工具链滥用（SSRF 到内部元数据）、跨租户 RAG 泄露、基于查询的模型提取等真实攻击手法；专家层的 5 个实验采用 Fernet AES 加密打包成 challenges/expert.enc，需玩家完成前两层并从维护者处获取密钥后本地解密。每个实验都配套「Learn — the fix」防御讲解，做到攻防成对。平台内置漏洞助手全部为确定性脚本，无需外部 API Key，可离线复现 Flag。玩家进度以 data/progress.json 持久化，无需数据库。界面提供三档积分、提示扣分、通关证书卡（SVG/PNG 下载并可分享至 LinkedIn/X）、玩家昵称锁定与首次引导教程，并适配移动端。
AI 安全教育目前缺乏像 WebGoat 之于 Web 安全的公开靶场，LLMVault 首次将 OWASP LLM Top 10 (2025) 全类别落地为可复现的动手实验，且用脚本化确定性助手避免了对付费大模型 API 的依赖，让教学与红队练习门槛显著降低；分层解锁 + 加密专家关卡的设计也是较少见的社区化玩法，适合安全培训、CTF 组织者以及 GenAI 团队做内部演练。

个股投研报告 Agent Skill ⭐ 137

把AI一键变成机构级投研分析师

Markdown (Agent Skill 指令)Python 3 (标准库)Claude Agent Skills 规范MCP (IBKR / Morningstar，可选)Web Search / Web Fetch
Equity Research Skill 是一个面向 Claude Code、Claude Desktop（Cowork）、Codex CLI 等 AI Agent 工具的 Agent Skill，能将『帮我研究一下某只股票』这类自然语言请求，自动转化为一份事实准确、来源可溯、结论明确的机构级九章投研报告，输出为中文。项目覆盖美股、港股、A股三大市场，对 A/H 双重上市标的默认执行溢价对比并分市场给出结论。工作流分为五步：确认标的、探测数据源并并行采集（行情、Morningstar、财报与卖方观点，工具缺失自动降级并标注）、数据对账与时间戳、按九章模板撰写、估值交叉验证并交付文件。九章结构涵盖一页速览、业务详情、竞争与护城河、管理层与治理、财务分析、多方法估值、分析师观点汇总、新闻与催化剂、投资结论。估值纪律要求至少三种方法交叉验证：反向 DCF 证伪现价、三情景概率加权 DCF、三要素/EPV（含护城河验证与 franchise 成长）、相对估值或 SOTP，所有计算由 scripts/dcf.py 执行，禁止心算，假设以 JSON 留档。结论通过预注册标定规则映射到买卖动作矩阵，保证同一组数字不会产出不同结论。技能主体为纯 Markdown 指令，配合仅使用 Python 标准库的估值脚本，可选接入 IBKR MCP、Morningstar MCP 与 docx 技能。
这是一个把『投研方法论工程化』的典型样例：它没有训练新模型，而是用一套结构化的 Markdown 提示词加一个确定性的 Python 估值脚本，把 LLM 容易失控的『主观判断+心算估值』问题约束成事实与判断分离、数据带时间戳、估值必须交叉验证、结论按预注册规则映射的可复现流程。对于想学习如何设计高质量 Agent Skill、或想用 AI 辅助严肃投资研究的用户，这个项目在流程设计、降级容错、跨 Agent 兼容性方面都有很高的参考价值。

AgentSmith 通用智能体框架 ⭐ 309

模型无关的AI智能体运行骨架

Bash (setup.sh / verify.sh)PowerShell 7+ (setup.ps1)Markdown 规则文件 (CLAUDE.md / AGENTS.md / GEMINI.md)GitClaude Code / Codex / Gemini / Cursor 等智能体运行时MCP 服务器与插件包生态
AgentSmith 是一个通用、模型无关的 AI 智能体运行框架（operating harness），旨在为 Claude Code、Codex、Gemini、Cursor 等能读取 CLAUDE.md / AGENTS.md / GEMINI.md 的编码智能体提供一套可移植、经过实战检验的操作规范。项目的核心理念是：一个智能体的最终效果，模型只占约 10%，其余 90% 来自围绕模型的规则、工具、记忆、护栏和反馈循环，而这套 harness 正是可复用的那 90%。它由作者在近 6 个月、数百次自主任务会话中打磨出来，是第五次迭代版本，剥离项目细节后沉淀为一个精简的通用核心加上可插拔的工作类型 profile（软件开发、DevOps、市场推广、文档撰写、数据处理、研究、设计、通用行政等）。使用流程分四步：通过 setup.sh 一次性安装，把通用 core 与所选 profile 组装成精简的 CLAUDE.md；每次会话只处理一个跟踪单元，按「计划→实现→验证→交付」推进；通过 verify.sh 跑真实检查，坚持「证据先于断言」，修 bug 需先写失败测试；上下文用到 25%-30% 时主动 handoff，写好召回提示，并把出错的教训固化回规则本身。安装方式提供三档：让智能体自己读 AGENT-INSTALL.md 自主安装、交互式向导、以及命令行参数直驱，支持全局 core + 项目 profile 的分层安装，并提供 Windows 下 PowerShell 7+ 的等价 setup.ps1。
在「vibe coding」满天飞的当下，AgentSmith 提出了一个稀缺视角：把 AI 编程的可靠性问题拆成模型与 harness 两层，并把 harness 这 90% 的工程实践产品化。它不是又一个 prompt 模板集合，而是一套被数百次真实自主会话打磨过的操作系统——包含验证闭环、上下文预算管理、失败即改规则的自省机制。对希望把 AI 智能体从 demo 推进到长期维护阶段的团队来说，这类可复用、模型无关的基础设施极具参考价值。

通用 Fable 5 引擎系统提示词 ⭐ 138

精简版跨模型通用系统提示词

MarkdownPrompt Engineering跨 LLM 系统指令 (ChatGPT / Gemini / Claude)
该仓库自称是「泄露」的 Claude Fable 5 / Mythos 5 系统提示词的精简重构版本。作者主张原始提示词长约 12 万字符、约 3 万 token，充斥着 Anthropic 专属的 XML 工具封装、安全覆盖和服务端函数，无法在其他模型上高效运行。项目通过剥离 Claude 专有基础设施（如 标签及环境专属指令），将提示词压缩至约 500 token，同时声称保留了原始架构的核心行为逻辑：自主推理、多轮时程规划、自我验证循环，以及所谓的「反聊天机器人」行为规则。仓库仅包含一份 system_prompt.md 文件，README 说明了在 Google AI Studio、OpenAI Developer Platform、ChatGPT 自定义指令以及 Gemini Gems 构建器中的部署方式。作者强调四个保留的架构杠杆：反聊天机器人的散文标准（避免堆砌项目符号和虚假热情）、结构化再解构（强制模型重建源数据）、禁止思考旁白（消除「让我思考一下」类元评论）、以及零互动陷阱（去掉「需要帮助请告诉我」类话术）。项目采用 MIT 许可，欢迎针对前沿模型的边缘用例提交 PR。需注意的是，README 中提及的模型版本（如 ChatGPT 5.6、Gemini 3.1 Pro）以及所谓「泄露」来源均无公开事实依据，读者宜谨慎甄别。
