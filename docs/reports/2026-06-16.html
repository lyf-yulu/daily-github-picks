Idea-to-Ship Skills 是一套面向 Claude 与 OpenAI Codex 的可组合 Agent Skills（智能体技能）集合，目标是把一个模糊的产品想法系统化地推进到验证、排期、构建和上线。项目把工作流划分为两个层级：手动层（manual tier）包含 ideate（构思发散）、deep-dive（深度调研）、prompt-pack（提示词包）三个技能，帮助开发者在人机协作下打磨想法；自动层（autonomous tier）包含 autopilot（自动驾驶）和 build-loop（构建循环）两个技能，让 Agent 自主执行需求拆解、代码生成与迭代验证。这种分层设计既保留了人工介入的灵活性，又能在思路成熟后切换到全自动模式持续推进交付。整体定位为大模型 Agent 的工程化能力扩展包，适合产品原型验证、独立开发者快速 ship 项目以及 AI 驱动的研发流水线场景。

Architect-Loop 跨厂商智能体协作骨架 ⭐ 454

Claude 当架构师，Codex 当工程师，仓库做记忆

Claude Code SkillAnthropic Claude APIOpenAI Codex / GPT-5 系列 APIGit 仓库作为状态存储Markdown 规约文档 (设计稿 / ADR / TODO)
Architect-Loop 是一个面向 Claude Code 的 Skill（技能扩展），目标是把两家厂商的旗舰编码模型组合成一条研究背书的智能体协作流水线。其核心设计哲学是角色分工：以 Claude（README 中标注为 Fable 5 系列）担任 architect，负责需求拆解、技术选型、接口契约和实施计划；以 GPT-5 Codex 担任 builder，负责把架构师产出的规格落地为可运行代码、补丁和测试。两者之间不直接共享上下文，而是通过 Git 仓库本身作为持久化记忆层进行交接：架构师把决策与计划写入仓库（如设计文档、TODO、ADR、测试规约），构建者读取这些产物并以提交、Diff、测试结果回写状态，下一轮架构师再基于仓库当前状态继续推理。这种 repo-as-memory 的模式回避了跨厂商 API 之间无法共享会话的问题，同时利用版本控制天然提供可回滚、可审计的轨迹。项目以 Skill 形式分发，意味着开发者可在 Claude Code 中一键加载该工作流模板，让 Claude 自动按既定协议调用 Codex 子代理，形成稳定的规划-实现-验证循环，适合中长期、多步骤的代码演进任务。
在多数 agent 框架仍困在单厂商单模型的当下，这个项目用极简的方式回答了一个真实痛点：如何让擅长规划的模型和擅长写码的模型协作，又不被会话隔离卡住。把仓库本身当作共享黑板是个朴素却有效的工程取舍，对想搭建跨厂商编码代理的团队有直接借鉴意义。

Agent Harness Generator 是一个面向 AI 代理的元级（meta-harness）脚手架工具，用于快速生成专属、品牌化、聚焦特定领域的代理运行框架。开发者可以通过它一键搭建带有独立 npx CLI 的代理项目，内置 MCP（Model Context Protocol）服务器、持久化记忆模块、学习反馈循环以及由 witness 签名的可信发布机制。该工具兼容当下主流的代理生态，包括 Claude Code、Codex、pi.dev、Hermes、OpenClaw，以及提供硬件隔离沙箱的 RVM 运行环境，方便团队在不同的代理后端之间复用同一套 harness 结构。其定位是为代理工程化提供标准化、可分发、可验证的项目骨架，降低从原型到可信发布的工程成本。技术栈以 Node.js / npx CLI 为入口，配合 MCP 协议进行能力暴露，并通过签名机制保障供应链可信。

Loop Engineering 工具集 ⭐ 265

面向AI编码代理的循环工程实践框架

PythonCLILLM AgentsPrompt Engineering
Loop Engineering 是一个面向 AI 编码代理（AI Coding Agents）的实践型工具集与模式库，旨在帮助开发者设计能够提示、编排并持续优化 AI 代理的系统化工作流。项目灵感来源于 Addy Osmani 与 Boris Cherny 在 AI 辅助编程领域的研究，将传统的「Prompt Engineering」升级为「Loop Engineering」，强调代理在持续循环中的自我反馈、审计与成本控制。仓库提供了三款核心 CLI 工具：loop-init 用于快速初始化一个具备循环结构的代理项目脚手架，包含预置的提示模板、目录约定与配置文件；loop-audit 用于审计代理执行过程中的提示链路、上下文使用与决策轨迹，帮助识别冗余调用与潜在的失控循环；loop-cost 则聚焦于多轮代理调用的 Token 与 API 成本估算，支持按任务、按模型维度统计开销，便于团队在生产环境中预算管理。除工具外，项目还沉淀了一套可复用的设计模式与启动模板（starters），覆盖任务分解、检查点、回滚、人机协同审批等典型场景，使开发者能够将零散的 Prompt 实验转化为可度量、可治理的工程实践。整体定位介于框架与方法论之间，既适合个人开发者搭建实验环境，也可为团队构建生产级 Agent 系统提供参考骨架。
在 AI 编码代理快速普及的背景下，多数项目仍停留在 Prompt 层面，而本仓库率先提出「Loop Engineering」的系统化视角，将代理运行的可观测性、成本与治理纳入核心设计。它不仅给出概念，还以 CLI 工具的形式落地了审计与成本两大刚需能力，对正在构建生产级 Agent 系统的团队具有直接的工程参考价值。
