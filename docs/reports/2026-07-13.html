AI/Agent 安全评估的 SKILL.md 技能包

SKILL.md 规范Markdown 参考文档Claude Code Plugin MarketplaceVercel skills CLI (npx skills)OpenAI Codex skill-installerShell 安装脚本
SAIL Skill 是 Pillar Security 推出的一款 Agent 技能包，将其 SAIL V2（Secure AI Lifecycle，安全 AI 生命周期）框架封装为可在多种 AI 代理平台上一键安装的技能。核心内容是一份覆盖 7 个生命周期阶段、共 91 项 AI/Agent 风险的完整目录，每条风险都配有缓解措施和主流合规标准映射（ISO/IEC 42001、EU AI Act、OWASP LLM/Agentic、DASF、AIUC-1）。技能被设计为按需读取的参考文件，让 Agent 在回答 AI 安全相关问题时，都能引用真实的 SAIL 编号与映射，避免凭空捏造控制项。功能上，它可以对某个 Agent 或系统做完整的差距分析、生成安全路线图与成熟度评估、输出合规检查表、优先级排序控制、跨团队对齐材料，以及供应商 RFP 问卷。项目同时是 Claude Code 的插件市场，在 Claude Code 中额外提供 /sail、/sail:assess、/sail:roadmap、/sail:comply、/sail:rfp 五个斜杠命令，支持一键运行智能默认工作流。安装方式覆盖 Claude Code、Claude.ai/Desktop/Cowork、OpenAI Codex CLI、Google Antigravity、pi、ChatGPT、opencode 等所有兼容 SKILL.md 规范的 Agent 环境，并针对 AI Agent 自主安装场景提供了详细的操作指引。
AI Agent 安全正处于工程落地的关键期，SAIL Skill 把一份体系化的 91 项风险目录变成 Agent 可以随时调用的"知识底座"，让任何兼容 SKILL.md 的 AI 助手都能一键具备企业级 AI 安全评估能力。它同时展示了 SKILL.md 作为跨平台 Agent 能力分发标准的潜力，一次编写、多端复用，对希望构建可复用 Agent 专业技能的团队具有很强的参考价值。

Claude Antigravity Agents 协作技能 ⭐ 102

让 Claude Code 委派任务给 Gemini 子代理

Claude Code SkillsGoogle Antigravity CLI (agy)Gemini 模型Markdown (SKILL.md)Git worktreeBash / PowerShell
Claude Antigravity Agents 是一个为 Claude Code 设计的技能包（Skill），核心是一份名为 antigravity-agents 的 SKILL.md 指令文件。安装后，Claude Code 从单一执行者升级为编排者（Orchestrator）：它负责撰写自包含的任务简报，通过非交互方式（agy -p）调用 Google Antigravity CLI 启动后台子代理，由 Gemini 等模型执行繁重工作，同时 Claude 继续处理当前任务，形成真正的并行开发流。项目主打三大价值：一是节省 Claude 令牌与上下文，将全仓审计、大规模重构、研究调研等重活外包；二是引入独立的第二模型（Gemini）提供交叉验证意见；三是通过并行 fan-out 提升整体吞吐。安全模型是设计核心，遵循「子代理绝不写入 Claude 正在编辑的文件」的铁律：只读任务如代码审查、架构分析、依赖审计运行在沙箱模式；写文件任务如编码与重构则在独立 git worktree 或副本仓库中执行。任何子代理返回的产出，Claude 都会通过 git diff、类型检查与构建流程进行验证后再合并，避免盲信。安装仅需克隆仓库并复制到 ~/.claude/skills 目录，Windows、macOS、Linux 均兼容。
该项目提出了一种务实的多代理协作范式：不同厂商的编码代理（Anthropic Claude 与 Google Gemini）通过 CLI 层松耦合协同，主代理专注编排与验证，副代理承担重活，既省令牌又能获得跨模型的独立第二意见。相较于纯粹的单模型 Agent 循环，它用「隔离 + 验证」两条工程化护栏解决了多代理并发中最棘手的文件冲突与信任问题，落地成本极低（一个 SKILL.md 文件），是 AI 编程工作流演进的一个具有参考价值的样本。

Ditto ⭐ 137

从AI编码日志挖掘你的工作画像

PythonClaude Code SkillsCodex Pluginnpx / skills.sh bootstrapCursor / Gemini 适配器
Ditto 是一个本地化的开源工具，它把你与 Claude Code、Codex、Copilot CLI 等 AI 编码代理的历史会话日志挖掘成一份私有的 you.md 工作画像，让新会话中的 AI 代理不再从零开始认识你。与传统的 memory 机制（你显式告诉模型的信息）不同，Ditto 读取的是原始会话日志而非 CLAUDE.md 或规则文件，从真实工作证据中提炼你的隐性规则：你如何定义『完成』、你会当场拒绝什么样的代码、你在何时要求提供证明、什么样的 agent 行为会让你叫停任务，以及你真正投入工作时的语言风格。挖掘结果按工作、设计、写作分层组织，确保针对不同任务加载合适的画像切片。工具运行完全本地化，敏感信息会先做脱敏处理，日志不离开本机。核心命令 python ditto.py --card 可生成一张可分享的画像卡片，包含 archetype、按不同会话证据排序的核心法则、覆盖统计和一句尖锐的自我认知。项目提供 npx skills add 一键 bootstrap，同时为 Codex 提供原生插件 ditto:mine / work / design / write，并通过适配器支持 Cursor、Gemini 等多家代理。运行前会先打印只读的成本计划等待用户批准，避免意外消耗模型额度。整个项目用纯 Python 实现，零依赖。
它把『AI 记忆』这个话题从显式提示转向了行为考古：不是让你再写一份规则文档，而是把你已经产生的会话数据视为最诚实的行为证据，反向挖掘出你自己都没写下来的工作准则。README 中的 Reddit 实测（168 赞、88K 浏览）提供了可验证的效果对比，加上零依赖 + 跨代理 + 本地脱敏的工程克制，让它在众多 AI agent 记忆项目里显得独具视角，也预示了 personal LLM profile 这一新兴方向的落地形态。

Fable-Mode 工作纪律协议 ⭐ 105

用工作纪律让弱模型逼近前沿输出

Claude Code SkillPython 3 (标准库)Bash 安装脚本Claude Code Hooks (SessionStart/PreToolUse/PostToolUse/Stop)Markdown (SKILL.md / SPEC.md / LEDGER.md)
fable5-mode 是一个面向 Claude Code 的 Skill 加钩子（Hook）套件，核心命题是「输出质量 = 模型能力 × 工作纪律」。它不试图让 Opus 4.8 变成 Fable 5 那样的前沿模型，而是把前沿模型隐含的「工作习惯」显式化并强制执行，从而在中低端模型上补齐流程层面的落差。项目提炼了六条纪律：计划闸门（先写 docs/SPEC.md 与可机器验证的验收条件）、小卡片执行（每张卡在全新上下文中运行，两次失败即升级）、对抗式自检（关键输出走 refute pass 或 N 方案+评审+合成）、真实产品验证（跑通端到端并留下截图/日志证据）、上下文卫生（SPEC/PROGRESS 作为外部记忆）、检查点自治（长任务加看门狗与可恢复检查点）。落地上，四个 Hook 把最常被偷懒的规则变成硬约束：SessionStart 注入纪律画像；PreToolUse 的 Spawn Guard 拦截未建 ledger 的分派以及越级请求更强模型；PostToolUse 在连续 3 次命令失败时注入归因阶梯；Stop 钩子在 ledger 有未完成项或已完成项缺少证据时阻止收尾。同时提供 fable_lint.py 做 SPEC 与证据的一次性检查。设计上按项目 opt-in（.fable/ 目录）、失败开放、循环安全、支持豁免，可安全全局注册。
它把「Prompt 工程」推进到「过程工程」——不是靠更花哨的提示词，而是通过外部账本、钩子拦截和证据校验，把模型难以自觉遵守的工作纪律变成可机械执行的流程。对于希望以低成本模型完成严肃工程任务、或研究 Agent 可靠性与自我校验机制的团队，这是一份少见的、可直接复用的工程化蓝本。
