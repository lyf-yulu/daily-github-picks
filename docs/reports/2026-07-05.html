token-diet ⭐ 456

为编程 AI 代理持续削减 token 消耗的效率插件

Shell（install.sh 安装脚本）Node.js（bench/bench.mjs 基准测试）Markdown / AGENTS.md 规则注入Claude Code SessionStart HookCursor / Windsurf / Cline Rules 文件Anthropic API（测试基准）
token-diet 是一个面向主流 AI 编程代理（Claude Code、Codex、Cursor、Windsurf、Cline）的 token 效率优化工具，以「始终在线」的方式注入到代理的工作流中，无需每次手动触发。其核心理念是在不损失正确性的前提下，系统性地压缩整个会话周期内的 token 消耗，涵盖回复、文档、测试、代码生成、上下文读取和工具调用等各个环节。 项目的工作原理是通过安装脚本自动检测当前使用的 AI 代理类型，并将一套「精简指令集」注入到对应的会话触发机制中：Claude Code 使用 SessionStart 钩子，Codex 使用 AGENTS.md 常驻上下文文件，Cursor/Windsurf/Cline 则通过规则文件实现常驻加载。这样每次会话启动时，精简策略即自动生效。 功能层面，token-diet 从六个维度进行优化：回复方面要求直接给出答案，去除「当然！以下是……」式的无效开场和结尾客套；文档与注释只保留解释「为什么」的关键信息；测试只覆盖关键路径和边界场景，上限 10 个/会话；代码遵循 YAGNI 原则，只实现被明确要求的功能；上下文读取优先 grep 精准定位，避免整文件读取；工具调用则批量合并独立请求并在信息充分时立即停止。 项目提供三档强度：默认 on 档全量启用、lite 档仅优化沟通与产物输出、ultra 档启用电报式极简回复风格。实测数据（基于真实 Sonnet 5 会话）显示：输出型会话（建议、规划、解释）token 输出减少 81%、账单降低 54%；代码修改+测试场景账单降低 22%；以读取为主的场景账单降低 17%；三类场景平均账单节省 31%，且正确性在所有测试中均未下降。
AI 编程代理的 token 成本在高频使用场景下已成为实际痛点，而 token-diet 提供了一种无侵入、跨工具链的系统级解决方案。它不依赖模型微调或代理私有接口，而是利用各代理现有的「会话初始化」机制注入约束指令，具备极强的可移植性和零维护成本。更重要的是，它将「简洁」的边界划定得非常清晰——仅作用于输出，绝不压缩推理过程和关键验证，这使其在工程实践中可信赖。对于重度依赖 AI 编程代理的团队或个人开发者，这是一个低风险、高回报的效率杠杆。

loopkit 是一个面向编码类 AI 智能体（Claude Code、Cursor、Codex、Gemini CLI 等）的技能集合与最小化运行框架。项目核心是 33 个经过实战打磨的 Skill 文件，分为 9 个赛道：智能体/LLM 工程、调试、安全、前端、测试、重构、文档、数据、Git/运维，另外附带一个专门用于对抗性验证的 review 赛道，覆盖智能体伪装『完成』时常见的 11 种偷懒模式。每个 Skill 都是一段聚焦的指令集，只有在触发条件匹配时才被加载，从而让智能体在具体场景下专业化响应，避免『每次提示都为所有能力付 token』的浪费，也降低了幻觉与虚构 API 的概率。除技能库外，安装脚本还会在目标目录写入一套最小 harness：60 行的 CLAUDE.md 常驻上下文、settings.json 权限白名单与写入即格式化钩子、verifier 对抗验证子智能体、MCP 服务器接线、跨会话 MEMORY.md 索引，以及 Plan→Act→Verify 循环执行脚本 run.sh。用户可通过 curl 一键安装，默认保留已有文件，可用 FORCE=1 覆盖。项目理念参考 Anthropic 的 Building Effective Agents 与 Claude Code 官方文档，将智能体工程模式蒸馏成可即插即用的规范。MIT 许可，鼓励 Fork 定制。
在多数智能体工具还停留在『大而全的系统提示』阶段时，loopkit 用触发式懒加载 Skill 的方式把提示工程模块化，配合对抗性 Verifier 与 Plan→Act→Verify 循环，直击当下编码智能体最痛的『虚构 API、假通过测试、改名当修复』问题。它把 Anthropic 官方的智能体工程理念做成了任何 CLI 智能体都能秒接入的落地方案，且完全 MIT 开源、可 Fork，适合作为团队自建 AI 编码规范的起点。

OrchestralFlow 智能体交响编排框架 ⭐ 156

用交响乐比喻编排 Claude Code 多智能体

Claude Code (Opus 系列).claude Manifest 声明式配置原生 Prompt 编排 (无 pip/npm 依赖)UltraCode Prompt 压缩
OrchestralFlow 是一个受 Claude Code 多智能体能力启发的声明式代理编排框架，其核心理念是把代码生成流水线类比为一支交响乐团：/goal 相当于指挥棒，Skills 是演奏技法，Subagents 是乐手，Agent Teams 是乐团分部。项目主打「成本阶梯」(Cost Ladder)——按子任务的复杂度将请求自动路由到不同层级的智能体：简单查询交给合奏智能体 Tutti，架构决策交给独奏智能体 Soloist，元推理与跨切面问题交给指挥智能体 Conductor，且每 5 个乐章重新评估一次。执行流程被组织为五个乐章：主题声明、技能映射(呈示部)、动态团队组装(展开部)、并行执行与反馈(再现部)、以及生成事后评分的尾声。项目宣称零依赖，仅通过单一 .claude 清单文件加载，并内置 UltraCode 压缩层，声称可以将 Prompt 令牌开销降低 40%。其余特性还包括长任务中的智能体热切换、多语言 /goal 解析、以及在成本超支 15% 时自动插入重构乐章的安全检查机制。
该仓库在概念表达上颇具巧思，用交响乐术语系统性地映射了 Skills、Subagents、Agent Teams 与成本控制之间的关系，对理解 Claude Code 多智能体分层与预算感知路由是一个有趣的思维模型。不过需要提醒的是：README 中提到的 Claude Code 版本号、40% 压缩率等数据未见实测依据，且页面顶部的 Download 按钮指向 github.io 外链而非仓库源码，实际实现是否与描述一致需读者自行核实，建议将其视为理念性作品而非可直接落地的工程框架。
