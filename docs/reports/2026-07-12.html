SAIL Skill 是 Pillar Security 推出的开源 Agent 技能包，将其 SAIL V2（Secure AI Lifecycle）框架封装为可跨平台调用的 AI 代理技能。项目将 SAIL V2 的完整风险目录内置为参考文件——覆盖 AI 生命周期 7 大阶段共 91 项风险，每项均附带缓解措施和标准映射，Agent 按需读取，从而保证输出的风险 ID、控制项、合规映射都来自真实目录而非幻觉。使用者只要在支持 SKILL.md 规范的代理里触发 AI/Agent 安全相关意图，技能便会自动激活，能够完成 Agent 或系统的完整 SAIL 差距分析、生成 7 阶段安全路线图与成熟度评估、按 ISO/IEC 42001、EU AI Act、OWASP LLM/Agentic、DASF、AIUC-1 等框架生成合规清单、输出跨团队协同产物以及厂商 RFP 问卷。项目同时是一个 Claude Code Plugin Marketplace，除通用 SKILL.md 触发外，还额外提供 /sail、/sail:assess、/sail:roadmap、/sail:comply、/sail:rfp 五条斜杠命令一键完成审计与合规。安装适配 Claude Code、Claude.ai/Desktop/Cowork、OpenAI Codex CLI、Google Antigravity、pi、ChatGPT、opencode 等主流环境，并预置分发压缩包和多路径拷贝方案，甚至专门给 AI 代理写了自安装指引，展现出对多 Agent 生态的高度兼容性。
在 AI Agent 大规模落地但安全评估工具稀缺的当下，SAIL Skill 把一份权威的 91 项风险目录做成可被任何 SKILL.md 兼容代理直接调用的技能，等于让 CISO 用自然语言就能对 LangGraph、MCP 等复杂 Agent 系统做出可审计、可对齐 EU AI Act/ISO 42001 的安全评估，是 Agent 安全治理领域少见的开箱即用参考实现。

Claude Antigravity Agents ⭐ 101

让 Claude Code 化身编排者，委派任务给 Antigravity 子代理

Claude Code Skill (SKILL.md)Google Antigravity CLI (agy)Gemini 模型Git worktree 隔离Bash / PowerShell 安装脚本Markdown 指令文件
Claude Antigravity Agents 是一个面向 Claude Code 的技能包（Skill），核心只有一个 SKILL.md 指令文件，命名为 antigravity-agents。它教会 Claude Code 将耗时或大范围的编码、审查、分析、研究类任务委派给 Google Antigravity CLI（agy）驱动的子代理执行。加载技能后，Claude Code 从单一执行者转变为编排者：先编写自包含的任务简报，通过 agy -p 在后台非交互式启动作业，同时继续处理用户当前的开发工作；子代理完成后，Claude 会读取输出，并通过 git diff、类型检查、构建等步骤对结果进行验证后再采纳。项目的价值主张有三点：一是节省 Claude 上下文与订阅 token，将重活外包给 Antigravity；二是引入 Gemini 等异构模型，获得真正独立的第二意见；三是通过并行 fan-out 多个子代理提升吞吐。安全模型是核心设计之一：只读类任务（代码审查、架构分析、依赖审计、研究扫描）在 sandbox 中运行，可与 Claude 并发；写文件类任务（编码、重构）则强制在独立的 git worktree 或独立仓库中执行，绝不与 Claude 当下正在编辑的文件冲突。安装方式极简，只需将 antigravity-agents 文件夹拷贝到 ~/.claude/skills/，Claude Code 会在下次启动时自动发现。支持 Windows、macOS、Linux 三端。
该项目用一份 markdown 文件就把 Claude Code 从单线程编码助手改造成多代理编排者，代表了一种轻量却极具想象力的 AI 工程范式：不改代码、不搭框架，纯靠 prompt 契约和 CLI 组合，就实现了跨厂商模型协作、并行作业与安全隔离。对于关心 token 成本、多模型协同或 agentic 工作流设计的开发者，这是一份可以立刻上手复用的最小可行方案。

Cognitive-Core-Skills 是一个通用、跨行业中立的认知核心技能分类体系（Taxonomy），旨在为大语言模型（LLM）、小语言模型（SLM）、AI 智能体（Agent）以及世界模型（World Model）提供一套统一的能力描述框架。项目把智能体的“操作大脑”拆解为八大核心能力：感知（perception）、记忆（memory）、推理（reasoning）、规划（planning）、行动选择（action）、验证（verification）、学习（learning）与治理（governance），并进一步细化为 159 张技能卡（Skill Card），覆盖 13 个领域，包括文本类技能、人类行动类技能、世界模型技能、持久化 LLM Wiki 知识运营、以及 OKF 风格的知识编目操作。仓库以工程化方式交付分类数据：提供 JSON/YAML 机读数据、JSON Schema 校验规范、Markdown 技能卡、评估评分量表（Evaluation Rubric）、认知负债指数（Cognitive Debt Index）以及基准任务轨迹（rubric-task-traces）。同时配套 GitHub Actions CI，运行 pytest 对分类体系与基准夹具进行一致性校验，并通过脚本 scripts/generate_skills.py 一键从数据源再生成技能卡文档，确保数据、Schema、文档三者始终同步。项目还提供贡献指南、Good First Issue 引导、发布检查清单和路线图，便于社区共建。适合希望构建可评估、可治理 AI Agent 能力体系的团队直接复用作为工程底座。
在 AI Agent 能力评估仍缺乏统一语言的当下，本项目提供了一套结构化、可校验、可扩展的认知能力分类框架，把“智能体到底会什么”从模糊描述变成 159 张标准化技能卡，并配合 Schema 与 CI 形成工程闭环，可直接作为 Agent 能力矩阵、评测基准和治理清单的底座，对研究者、平台方与企业落地都有明确参考价值。

Ditto ⭐ 106

挖掘编码日志生成你的Agent画像

Python零依赖Claude SkillsCodex Pluginnpx / skills.sh 引导
Ditto 是一个开源的本地工具，用于从 Claude Code、Codex、Copilot CLI 等编码 Agent 的会话日志中挖掘用户真实的工作习惯，生成一份私有的 you.md 工作画像。它的核心理念是：Memory 只记录你显式告诉模型的内容，而 Ditto 挖掘的是你的工作已经证明的东西——你会否决什么、如何定义完成、何时要求证明、怎样调试、如何设计 UI，以及在真正工作时怎样表达。工具会读取原始会话日志（而非 CLAUDE.md 或规则文件），剥离工具输出、错误信息与文件转储，只保留用户本人的语言，然后分块交由 Agent 解析，提炼出可被日期化会话回执支撑的行为法则，最后合并为一份可供 Agent 每次任务前加载的画像。Ditto 支持工作、设计、写作三个分层，让不同任务加载对应侧面。运行时会先输出只读的挖掘计划，展示成本并等待用户批准；处理过程完全本地化，敏感信息在处理前会被脱敏。命令行支持 --card 生成可分享的档案卡片，展示原型、按证据排序的法则、覆盖统计与一句尖锐真相。安装方式支持 npx skills 跨 Agent 引导脚本，也提供原生 Codex 插件（ditto:mine / work / design / write），采用 Python 且零依赖。
Ditto 提出了一个反直觉但有说服力的观点：真正定义你工作方式的不是你写下的规则，而是你从未写下的会话痕迹。作者用一个真实案例（Reddit 帖子从冷启动到 168 赞、88K 浏览）证明了画像加载对 Agent 输出质量的量变。相比 Cursor Rules、CLAUDE.md 这类显式记忆方案，Ditto 走出了一条以行为证据为核心的差异化路径，且完全本地、零依赖、开源 MIT，对关心 Agent 个性化与隐私的开发者极具参考价值。

fable-mode ⭐ 105

让弱模型习得强模型工作纪律

Claude Code SkillPython 3 标准库Bash 安装脚本Markdown (SKILL.md/SPEC.md/LEDGER.md)JSON (settings.json Hooks 配置)
fable-mode 是一个基于 Claude Code 的技能（Skill）加守卫钩子（Hooks）的工作纪律协议，核心命题是「输出质量 = 模型能力 × 工作纪律」。它试图通过强制性的流程约束，让 Opus 4.8 等非前沿模型也能以接近 Fable 5 级别的严谨度完成任务。项目围绕六条杠杆展开：计划门（先写 SPEC.md 与可机器校验的验收测试）、小卡片执行（每张卡片独立上下文，两次失败即升级）、对抗式自检（派发独立视角反驳关键输出）、真实产品验证（端到端跑通并留下证据）、上下文卫生（用外部 SPEC/PROGRESS 代替臃肿会话）和检查点自治（长任务带看门狗与断点续跑）。为避免仅靠 Prompt 靠模型自觉，项目提供四个 Hook：SessionStart 的 Profile Injector 按 Ledger 状态注入纪律，PreToolUse 的 Spawn Guard 拦截无计划派生与越级模型请求，PostToolUse 的 Fail-Streak Reminder 在连续失败时提示归因阶梯，Stop 的 Close Guard 禁止在未完成条目或缺证据时结束回合。此外还附带 fable_lint.py 用于 CI 检查 SPEC 标注与证据完整性。整套设计以 .fable/ 目录按项目 Opt-in，Fail-open 且循环安全，可全局注册而不影响未启用的项目。
它把「AI 工程化」从口号推进到可执行层：不再依赖模型自律，而是通过 SessionStart/PreToolUse/PostToolUse/Stop 四类 Hook 把纪律固化为运行时约束。这种「用协议弥补能力差距」的思路为如何低成本使用非前沿模型提供了可复用范式，对 Agent 系统设计和团队级 AI 规范制定都有直接借鉴价值。
