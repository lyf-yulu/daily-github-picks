Claude 当架构师，Codex 当建造者

Claude Code (Claude Fable)OpenAI Codex CLI (>=0.133)Bash / PowerShell 安装脚本git worktreeMarkdown skill 规约Python 校验脚本
architect-loop 是一个面向 Claude Code 的技能（skill）包，把 Anthropic 的 Claude Fable 与 OpenAI 的 GPT-5.5 Codex 拼成一个跨厂商的智能体闭环：Fable 负责规划与评审，Codex 负责实现与调研，仓库本身则承担长期记忆。项目通过两个斜杠命令暴露能力：/architect 触发构建循环，由 Fable 把任务切成单 PR 切片并提前把验收门槛写入 docs/gates/，再分派 1–4 条 lane 到独立 git worktree 中的 codex exec 实例并行执行；建造者无提交权，Fable 在新会话中亲自跑门控命令、对照 spec 审 diff，再合并通过的 lane。/architect-research 则是研究循环，先用便宜的 Codex 侦察员扫描主题，再由 Fable 按主题设计 3–6 条带预算约束的 lane 并行检索，最后做对抗性证伪、双源核验并撰写带引用的决策报告。设计上强调几条经验：弱规划比弱执行更致命、frozen 外部门控胜过信任 agent、记忆文件易腐烂所以 HANDOFF 只做短目录、生产级深度研究系统采用规划者动态分解而非固定 lane。安装只需克隆仓库执行 install 脚本并全局安装 Codex CLI，依托现有 Claude 与 ChatGPT 订阅运行，默认无需 API Key。
它把当下两类最强代码模型的优势用一种可复现的工程化方式拼起来，并把多智能体研究里被反复验证的几条原则（规划者强于执行者、外部冻结门控、新会话评审、规划器动态分解）落到具体的 skill 文件和门控命令里，是难得一份既有理论引用又能 30 秒上手的多 agent 协作模板。

MetaHarness 智能体脚手架工厂 ⭐ 118

为任意仓库生成专属 AI 智能体框架

Node.jsTypeScriptnpx CLIMCP (Model Context Protocol)Claude Code / OpenAI Codex / Hermes / OpenClawGitHub ActionsGitHub Pages (Studio Web UI)RVM 微管理程序
MetaHarness（agent-harness-generator）是一个用于生成定制化 AI 智能体框架的「元脚手架」工具，由 ruvnet 开发并以 MIT 协议开源。它的定位不是又一个智能体框架，而是「生产智能体框架的工厂」——给定一个 GitHub 仓库地址或空白起点，它能在 60 秒内、在浏览器或终端中本地生成一套与目标仓库高度契合的智能体工具链，包括仓库感知的 CLI、编码智能体、本地 MCP 服务器、项目级记忆命名空间、基于实际文件结构生成的技能、治理策略、发布闸门以及证人签名的产物溯源。生成结果是一个可直接 npm 发布的 zip 包，带有用户自己的品牌和 npx CLI 入口。项目支持九大宿主环境，包括 Claude Code、OpenAI Codex、pi.dev、Hermes、OpenClaw、RVM 硬件隔离沙箱、GitHub Copilot、OpenCode 以及 GitHub Actions CI/CD。MCP 作为一等适配层接入，采用默认拒绝策略，提供 off/local/remote 三种模式并自带 mcp-scan 静态扫描器（类比「npm audit for agent tools」）。配套的 @metaharness/router 还能基于自有评测数据将每次请求路由到「足够好的最便宜模型」，配合 DRACO 基准测试证明小模型可以以约十分之一成本达到前沿质量。项目内置 19 个垂直模板覆盖编码、DevOps 等场景，并提供 score、analyze-repo、wizard 等子命令，全程仅做静态分析、不执行仓库代码。
它跳出了「再造一个 Agent 框架」的内卷思路，把智能体框架本身作为可被生成、可被品牌化、可被组织内分发的产物，并把安全（默认拒绝 MCP、静态扫描、证人签名）与成本（智能路由 + 小模型基准）作为一等公民。对希望在团队/组织内规模化落地仓库感知型 AI 智能体的工程团队，这是一个少见的同时解决「定制化、安全治理、成本控制」三难问题的工程化方案。

Omnigent 元代理框架 ⭐ 2,794

统一调度多种 AI 编码代理的元框架

Python 3.12+uv / pip / Homebrew 分发Node.js 22 LTS（Claude/Codex/Pi harness CLI）tmux（终端复用）bubblewrap (bwrap) / macOS seatbelt 沙箱YAML（自定义代理定义）Modal / Daytona / Islo 云沙箱Web UI（本地 6767 端口）可选 Databricks 集成
Omnigent 是一个面向 AI 代理的「元 harness（元外壳）」开源项目，旨在为 Claude Code、Codex、Cursor、Pi 以及用户自定义代理提供统一的抽象层，让开发者可以在不重写代码的前提下自由切换或组合不同的代理 harness。项目核心定位是把「编码代理的运行时」标准化：通过 YAML 即可定义自己的代理，再交由 Omnigent 统一管理会话、终端、文件与子代理状态。它在跨设备协作上着力很深，会话可以从终端起步，无缝迁移到浏览器或手机端，团队成员还能加入同一活动会话进行实时观摩、共同驾驶或 fork 出独立分支。安全治理方面提供 policies 机制，可在风险动作前要求人工审批、限制工具访问范围或设定花费上限，并通过 macOS seatbelt 与 Linux bubblewrap (bwrap) 对每个代理终端做 OS 级沙箱隔离。运行环境方面除本地外还支持 Modal、Daytona、Islo 等云端一次性沙箱，便于无本地依赖地跑代理。模型接入层兼容 Anthropic / OpenAI 官方 API key、Claude / ChatGPT 订阅以及任意兼容网关，并可选 Databricks 工作区作为模型提供方。仓库附带 Polly（多代理编码编排者，将任务并行下发到不同 vendor 的子代理并交叉评审）和 Debby（双头脑暴 + /debate 互辩）两个示例代理，演示编排与多模型协同的典型用法。
在 AI 编码代理生态高速分化的当下，Omnigent 不绑定单一 vendor，而是把「代理本身」抽象为可插拔单元，并把多代理协作、跨设备会话、策略治理与沙箱隔离做成一等公民。它解决了开发者在 Claude Code、Codex、Cursor 等工具间来回切换、难以编排互评、缺乏统一安全边界的真实痛点，对探索多代理工作流与团队化 AI 协作有较强工程参考价值。

Harness Forge ⭐ 60

让 Claude Code 自我进化脚手架的原生技能

Claude Code SkillClaude Code Workflow / AgentPythonJavaScript (workflow 脚本)Markdown (SKILL.md / references)
Harness Forge 是一个 Claude Code 技能（skill），将 Meta-Harness（Lee 等人 2026）方法原生化重实现，用于在模型权重固定的前提下，自动优化围绕模型的脚手架——包括记忆、检索、上下文构造、摘要、提示模板与工具选择逻辑。它通过 propose（提出候选）→ score（打分）→ Pareto（保留帕累托最优前沿）→ repeat 的端到端循环来演化 harness 代码，质量上升、成本下降，并尊重质量底线。相比原始 Python 参考实现约 1260 行的 claude_wrapper.py 与 meta_harness.py，Harness Forge 借助 Claude Code 原生的 Agent、Workflow、parallel 等能力，将外层编排压缩到约 75 行，仅保留不可省略的 0 成本确定性打分器作为核心领域逻辑。项目提供一键安装脚本、插件市场入口、可运行的 memory-summary 示例，以及帕累托前沿计算脚本，并强调防止 frozen-replay 缺陷、抗 Goodhart 底线与 held-out 评估纪律。

鲁班（Luban）是 LearnPrompt 出品的一个 Agent Skill 打磨工坊，定位是把'自己能用'的 Skill 升级为'能被理解、能被安装、能被传播、能被验证、能持续进化'的公共资产。它面向已经写过 Skill 但发布后无人使用、缺乏可复现证据、不知如何优化的开发者，提供一套结构化的审视与改写工作流。鲁班的核心方法论由五个动作构成：验料（挑战 Skill 是否值得存在）、访行（联网对标同行、看清生态位）、过尺（用结构、实测、活体三把尺打分）、慢刨（冻结基线、单提交单面、过验证门才保留）、回炉（发布后留回炉清单，从反馈进入下一轮）。它以 Claude Code Skill 形式分发，可通过 npx skills add 或 plugin marketplace 安装，对话触发后会先输出一份带 URL 的同行对标和三把尺评分，并在用户授权前停手不改一字。最终交付包括一份 13 节《打磨报告》、一张可截图的'出师证书'，以及沉淀进仓库的验证脚本与项目规矩。实战战绩是把 ai-news-radar 从 v0.6 打磨到 v0.7.0，揪出 Actions 全绿下静默停更 8 天的数据链路，用 83,725 条历史数据回放清除 327 条假 AI、零误伤。
在 Agent Skill 生态野蛮生长、人人都能写但少有人能让自己 Skill 被装、被复用的当下，鲁班用'工坊师傅'的隐喻把抽象的'打磨'具象成可执行的五步流程，并显式吸收了 hv-analysis 的横纵分析、darwin-skill 的棘轮评测、microsoft/SkillOpt 的 validation-gated 编辑等前沿方法论，形成一个有方法论传承、有真实战绩、有授权边界的 meta-skill。它对任何想把 prompt 工程或 Agent 工具沉淀成公共资产的团队都具备直接借鉴价值。
