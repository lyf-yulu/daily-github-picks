面向AI编码代理的性能优化框架

该项目是一个面向AI编码代理（Agent Harness）的性能优化系统，旨在为 Claude Code、Codex、Opencode、Cursor 等主流 AI 编程助手提供统一的能力增强层。项目从五个维度切入代理的运行时表现：Skills（技能）、Instincts（直觉）、Memory（记忆）、Security（安全）和 Research-first Development（研究先行开发范式）。通过 Skills 模块，代理可以调用预置或自定义的能力组件；Instincts 模块负责在缺乏明确指令时给出合理的默认决策，减少反复试错；Memory 模块解决跨会话上下文丢失问题，让代理具备长期记忆；Security 模块在代理执行敏感操作前引入约束和审查机制；Research-first 理念则强调在写代码之前先阅读、检索、验证，避免盲目编码带来的返工。该系统定位为「跨 harness 的通用增强层」，而非绑定单一 IDE 或模型，因此可以在不同代理运行环境之间迁移复用配置和策略。目前项目主要以理念和方法论呈现，README 内容较为概要，具体实现细节需在仓库源码中进一步查阅。整体来看，它试图把「如何让 AI 代理更聪明地写代码」这一模糊问题工程化，转化为一组可组合、可复用的模块。
随着 AI 编码代理进入生产环境，单纯依赖模型能力已经不够，如何构建代理侧的记忆、安全与决策框架成为下一阶段的核心竞争点。该项目跨 harness 抽象通用能力，思路契合当前 agentic coding 的发展趋势，对研究代理工程化的开发者具有参考价值。

RealmForge 是一个开源的生成式智能体群（Agent Swarm）编排平台，旨在构建具备持久化状态与叙事连贯性的模拟世界。项目定位介于游戏 NPC 原型、世界观构建工具与 LLM 社会行为研究平台之间，核心理念是让每个由 LLM 驱动的角色拥有独立记忆、目标与人际关系，并在跨会话的共享世界中演化，从而涌现出未经预设脚本的故事线、经济系统与社会结构。其核心创新是结构化世界模型（SWM）——一个由实体（角色、地点、物品、概念）构成的有向图，智能体通过感知半径（Perception Radius）读取与写入图状态，从而实现信息不对称与探索机制。系统在此之上构建了三层架构：Conductor 编排层负责基于 Tick/Epoch 的调度与 REST API 暴露；Cortex Pool 负责 LLM 提示装配、模型路由（从 llama.cpp 到云端 API）与上下文压缩；Index 持久层则将世界图、时间序列的 Chronicle 事件日志、以及向量化记忆存储起来以支持 RAG。此外，项目还提供叙事市场（Narrative Market）作为剧情涌现引擎、多语言人格支持、以及 TUI Reactive Console 与 Webhook 子系统用于实时观测与外部集成。需要提醒的是，仓库名 mirage-cortex 与 README 描述的 RealmForge 不一致，且 README 中包含指向 GitHub Pages 的下载按钮，实际代码可用性与项目真实性建议在克隆前进一步核实。

self-learning-skills 是一个面向 AI 编码代理（Claude Code、Cursor、Codex、Aider、Zed、Gemini CLI 等）的自我改进型技能包。它解决的核心痛点是：每次会话中辛苦调试出的操作路径（如生产数据库连接方式、密钥位置、部署命令、验证步骤）会随会话结束而消散，下一次会话又需从零重学。该项目提供一个元技能（meta-skill），教会代理识别刚刚获得的可复用『黄金路径』时刻，并将其固化到工具能自动加载的位置，让下次会话直接掌握流程而非重新摸索。工作循环分三步：识别时机（多次尝试才成功的任务、非显而易见的命令、易复发的运维流程）、无需提示自动捕获（记录步骤与失败教训）、下次自动复用。项目按工具差异化落地：Claude Code 写入 skills//SKILL.md，Cursor 落到 .cursor/rules/learned/，其他基于 AGENTS.md 的代理则追加到标准指令文件。它还设计了分级路由（技能/备忘/丢弃）与晋升规则（需通过验证、命名失败模式、至少排除一条死路），避免把未经证实的猜测固化。安全上明确不写入任何密钥值，仅记录密钥所在位置。基于开放的 Agent Skills 标准构建，可通过 npx skills 或 Claude Code 插件市场安装。
这是把『AI 代理的经验』本身当作一等公民来工程化的思路：不解决具体任务，而解决『经验流失』这一元问题。它用极轻的形态（几个 Markdown 模板加安装脚本）覆盖了主流 AI 编码工具生态，并用『晋升规则』这样的社区智慧防止知识库劣化。对于团队协作场景尤其有价值，被采纳的黄金路径可随 git 共享，让整个团队的 AI 助手共享同一套已验证工作流，是可复用的 AI 记忆工程模式范例。
