Error Discovery Skill 是一个面向 AI 代理的交互式错误分析技能包，旨在帮助开发者系统化地排查 LLM 输出或代理轨迹中的失败模式。用户只需将 JSONL/CSV/JSON 格式的 LLM 输出数据集交给代理，它就会自动识别内容类型（如文章、代理轨迹、代码、结构化输出等），并基于格式塔原则设计视觉编码方案（用颜色区分类别、用间距体现层级、用透明度突出重要性）。随后，技能会用 Python 标准库启动一个零依赖的单文件 HTML 审阅应用，对数据进行聚类并挑选包含聚类代表和随机样本的多样化初始集合。在用户阅读样本并留下自由文本笔记的同时，代理持续监控标注、归纳失败模式分类、追踪覆盖度，并主动提议新的样本以填补盲区。整个技能以 SKILL.md 形式分发，任何能读取文件的代理（如 Claude Code）都能加载并执行。

OpenTag ⭐ 295

开源版 @agent 提及，工作流中召唤 AI

OpenTag 是一个开源的「@agent 提及」协议与运行时，让开发者在 GitHub、Slack 等工作工具中通过 @ 提及方式调用 AI 代理（如 Claude Code、Codex 或自定义执行器），并将结果回传到原始线程，形成可审计的闭环。其设计灵感来自 Claude Tag，但坚持开源与本地优先：项目不再把上下文复制到独立的 AI 工作区，而是把 Agent 带到团队真正发生工作的会话里。架构核心由轻量调度器（dispatcher）、运行器（runner，含本地守护进程 opentagd 与托管运行器）、执行器（executor，如 claude --print、codex exec）以及双向适配器（GitHub、Slack，实验性支持 Telegram 与 Lark）组成。每次 @ 提及会被规范化为一次有作用域的 Work Request，调度器负责持久化、租约、审计与回调；运行器仅认领显式绑定的 Project Target，执行器在隔离分支 opentag/ 上工作，并具备脏工作区保护。模型返回的建议动作（标签、Review、后续运行、PR 提交等）可在源线程通过 approve/apply/reject/continue 等回复指令进行审批，形成 Apply Plan 或子运行回退。当前 v0 版本已支持 GitHub Issue/PR 评论闭环、Slack 线程回调以及完整的协议运行时，多租户托管控制平面仍在演进中。
OpenTag 抓住了一个正在兴起的交互范式：在工作发生的地方 @ 提及 Agent，而非进入独立的 AI 聊天工作区。它把这个范式做成了一套开源、本地优先、可审计的协议（Work Thread、Context Packet、Audit Trail、Apply Plan），并且对外部写操作设置了显式的权限与审批门槛。对希望在企业内自托管 AI 代码代理、又担心权限与审计失控的团队，这是一个少见的、贴近真实工程治理的开源参考实现。

y 是一款面向 macOS（Apple Silicon）的桌面级编码代理应用，以聊天为主交互界面，将 Claude Code 与 OpenAI Codex 等本地 CLI 代理统一编排在一个可塑形工作区中。其核心理念是让软件在使用过程中保持可塑性：通过名为 Modify 的受保护改造通道，用户可以让 y 实时修改自身界面，生成的 UI 变更会经过编译、渲染和 diff 审查，确认无误后保留，渲染异常则回滚。架构上分为受保护的 Kernel（负责认证、引擎适配、文件系统边界、终端桥接、Modify 通道与回滚机制）和可变的 Userland（聊天 UI 与可被改造的应用表层），既实现自我修改又锁定信任边界。功能上支持 Claude Code 与 Codex 并行运行、隔离工作区下的多代理协作、文件树/终端/diff 视图，以及基于快照的回滚。隐私方面，聊天内容、项目文件与终端命令均不上传，仅收集基础产品分析数据。
