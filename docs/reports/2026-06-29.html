awesome-gamedev-agent-skills 是一个面向 AI 编程代理（AI coding agent）的游戏开发技能集合，包含 66 个原创、版本锁定的「Agent Skill」加一个主路由器（master router）。这些 Skill 采用便携的 SKILL.md 格式，可在 Claude Code、Cursor、Codex、GitHub Copilot、Gemini CLI、Windsurf、Cline、Kiro 等数十种 AI 编程工具中原生加载，无需格式转换。项目的核心理念是「按需加载」：开发者用自然语言描述需求（如「给我的 Godot 角色加二段跳」），路由器会自动识别项目所用引擎（通过 project.godot、*.uproject 等指纹文件判断）和任务类型（概念、类型或发布环节），再按「引擎基础 → 概念 → 类型黏合」的顺序仅加载相关 Skill，避免上下文污染。技能覆盖 10 大引擎生态，包括 Godot（15 个，针对 Godot 4.x）、Unity（8 个，针对 Unity 6 LTS）、Unreal、Phaser、PixiJS、three.js、Bevy、pygame、LÖVE、Roblox，并涉及 2D/3D 移动、瓦片地图、物理、动画、Shader、多人联机、ScriptableObject、导航、存档系统、程序化生成、平台发布（itch.io butler、Steam）等横向主题。每个 Skill 均基于一手官方文档编写并通过 validate-skills.py 校验，配合 npx skills add 一键安装或 Claude Code 插件市场分组安装，可按引擎选择最小子集。
它把「AI 代理 + 游戏开发」这个组合做成了可工程化的产物：通过路由器机制解决了 AI 编程工具在多引擎、多任务场景下上下文爆炸与知识漂移的痛点，且通过版本锁定与一手文档来源保证了技能的可信度，是当前少数面向具体垂直领域（gamedev）做 Agent Skill 标准化的开源参考实现，对希望构建自有 Skill 体系或评估 AI 辅助游戏开发可行性的团队具有直接借鉴价值。

错误发现技能 ⭐ 82

AI 智能体交互式错误分析技能

Python 标准库HTML/CSS/JavaScriptMarkdown (SKILL.md)JSONL/CSV/JSON 数据格式
Error Discovery Skill 是一个面向 AI 智能体的交互式错误分析技能包，让 Claude Code 等支持读取 Skill 文件的智能体能够对 LLM 输出或 Trace 数据集执行系统化的错误分析。用户只需将 JSONL/CSV/JSON 数据集交给智能体，它就会自动识别数据内容类型（文章、代码、智能体轨迹、结构化输出等），并基于 Gestalt 视觉设计原则（颜色区分类别、间距体现层次、透明度表达重要性）设计编码方案。随后，它使用 Python 标准库启动一个零依赖的本地 HTTP 服务，渲染单文件 HTML 审阅界面。系统会对数据进行聚类，挑选聚类代表加随机样本作为初始集，并进入交互循环：监控用户的自由文本标注、归类失败模式、追踪覆盖度，并主动提出新样本以填补空白。整个技能以一份 SKILL.md 指令文件交付，可放入 ~/.claude/skills 目录或复制到任意智能体的提示中使用。

OpenTag ⭐ 353

在Slack/GitHub里@唤起本地编码Agent

Node.js 20+TypeScriptpnpm monorepoSlack Socket Mode/Events APIGitHub WebhooksLark/Feishu APISQLiteCodex CLIClaude Code CLI
OpenTag 是一个开源的 @agent 提及（mention）路由工具，让团队可以直接在已经使用的协作平台（Slack、GitHub、Lark/飞书，以及实验性的 Telegram）中通过 @opentag 召唤本地运行的编码 Agent。当用户在频道、Issue 或评论中提及 OpenTag 时，它会监听平台事件，将请求分发到本地调度器（dispatcher）和运行器（runner），调用 Codex 或 Claude Code 在用户自己的电脑上执行任务，然后把结果回帖到原线程中。整个 CLI 流程采用 local-first 设计，没有 OpenTag 云服务，平台凭证以受限权限保存在本地 ~/.config/opentag/config.json，运行状态和隔离 worktree 存放于 ~/.local/state/opentag，代码 Agent 直接基于本地 checkout 工作。安装非常轻量，只需 Node.js 20+，通过 npm i -g @opentag/cli 或 npx 一键完成，opentag setup 会以四个问题引导用户选择监听地址、编码 Agent、本地项目和平台凭证，随后 opentag start 即可运行；另有 status、doctor、platforms、executors 等命令辅助诊断。项目按 monorepo 方式拆分了 cli、local-runtime、core、client、slack、github、lark、runner、store、dispatcher 等多个 @opentag/* 包，结构清晰，便于扩展新的平台适配器和 Executor。
把 ChatOps 与本地编码 Agent 结合，填补了云端 Copilot 与本地 CLI Agent 之间的协作鸿沟。团队成员可以像 @某人 一样把 Issue 或讨论直接派单给 Codex/Claude Code，而代码执行完全发生在开发者本机，兼顾了协作效率和代码安全。架构清晰、包拆分合理，为想自建 AI Agent 工作流或扩展更多平台/执行器的团队提供了可参考的开源范本。
