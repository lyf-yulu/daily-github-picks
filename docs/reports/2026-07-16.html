MeetingCopilot 是一款面向 Windows 与 Apple Silicon macOS 的实时会议 / 面试副驾工具，核心定位是在不加入会议机器人的前提下，直接抓取对方声音，实时转录并借助大模型生成第一人称的口播式答案，形成 Teleprompter（提词器）体验。Windows 端通过系统 Loopback 抓取音频零配置即可使用，macOS 端则借助 BlackHole 等虚拟音频设备完成路由，同时独立通道转录用户自己的麦克风。项目支持 4 种可切换的 ASR 后端：本地 FunASR 流式（默认，自动拉起 Python 子进程）、本地 Whisper turbo（DirectML GPU 离线兜底）、阿里云 fun-asr-realtime、以及 MiMo 分段识别，具备中英双语自动切换、灰色 partial 字幕、行内翻译等能力。答案生成层采用 BYOK 模式接入任意 OpenAI 兼容 LLM（推荐 DeepSeek），支持按会话导入简历与 JD，本地解析不上传，并针对行为题使用 STAR、技术题使用「思路→要点→复杂度」结构，配合滚动式面试备忘录保持长会话一致性。此外还提供前缀缓存预热、区域截图问答（MiMo / Gemini 视觉模型）、防录屏/防截屏隐身窗口、全局隐藏热键、深浅主题与中英文界面独立切换等实用功能，整体面向 1:1 高强度技术与行为面试场景。
该项目把「面试作弊器」这类灰色刚需产品做成了工程完成度很高的开源实现：一方面用本地 FunASR + BYOK LLM 的架构规避了服务端成本与隐私风险，另一方面通过前缀缓存预热、滚动备忘录、问题类型识别等细节把端到端延迟压到亚秒级，并额外解决了屏幕共享隐身、双语无缝切换、区域截图视觉问答等实际使用中的痛点，是研究实时 ASR + LLM 流水线与 Electron 隐身窗口技术的高质量参考样本。

deja-vu：编码代理的记忆层 ⭐ 199

让 AI 编程助手记住历史会话

GoMCP (Model Context Protocol)JSONL / SQLite 日志解析本地倒排索引 (records.bin + token buckets + manifest.json)sqlite3 CLI (opencode 索引)npm / Homebrew / Go module 分发
deja-vu 是一款面向 Claude Code、Codex CLI、opencode 等编码代理的本地记忆层工具，以单一零依赖二进制形式发布。这些代理会将每次对话写入本地日志文件，累计可达数 GB，deja-vu 通过解析 JSONL 与 SQLite 日志、构建本地倒排索引，将这些历史转化为可检索、可召回的记忆层。核心能力包括：毫秒级全文检索（3.3GB 语料中典型 7–9 ms）、通过 MCP 协议向代理暴露 recall 与 recall_context 工具、SessionStart 钩子实现自动上下文注入、索引期凭据脱敏（AWS Key、JWT、PEM、Bearer Token 等替换为 [redacted:]）、使用统计（deja stats）、会话分享（deja share 输出脱敏摘要）、以及基于共享目录或 SSH 的多机同步（append-only、幂等、带水位标记）。安装方式覆盖 curl 脚本、go install、npx、Homebrew，通过 deja install --all 可自动配置各代理的 MCP 与钩子并保留 .bak。索引存放于 ~/.cache/deja，采用增量策略，仅重新读取增长的会话文件，索引体积约为语料的 2.4%。所有数据不出本机，符合本地优先与隐私安全原则。
编码代理的对话历史通常是被忽视的高价值资产：里面沉淀着团队的调试路径与设计决策。deja-vu 用一个零依赖二进制把这些散落日志变成可被代理主动召回的记忆层，跨 harness 打通、脱敏、可同步、可分享，切中了 AI 编程工作流中的真实痛点，且工程实现克制、性能扎实、隐私友好，是当前 MCP 生态里少见的实用型基础设施项目。
