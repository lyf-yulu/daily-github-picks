token-diet ⭐ 610

编码智能体的常驻省 Token 技能包

BashShell 安装脚本Node.js（基准测试）Markdown 规则文件Claude Code HooksCodex AGENTS.mdCursor/Windsurf/Cline Rules
token-diet 是一款面向编码类 AI 智能体的常驻式 Token 节流技能包，支持 Claude Code、Codex、Cursor、Windsurf、Cline 等主流工具。项目目标是在不牺牲正确性的前提下，压缩整个会话周期内的 Token 消耗，覆盖回复、文档、测试、代码、上下文读取和工具调用等全链路环节。根据作者提供的 Sonnet 5 实测数据，输出量平均下降约 53%，账单平均节省约 31%，最高可达 54%（依会话类型不同波动于 -17% 到 -54%）。使用方式极为简洁，一条 curl 命令即可自动检测并安装到对应智能体，支持 on（默认）、lite（仅精简沟通与产物）、ultra（电报体聊天）、off 四档模式，并可通过 /token-diet 命令动态切换。技术实现上，作者利用了各智能体的常驻加载通道：Claude Code 的 SessionStart 钩子、Codex 的 AGENTS.md、以及 Cursor/Windsurf/Cline 的规则文件，使指令在每次会话启动时自动注入。核心策略包括：先答后释、去除寒暄套话、grep 优先于全文读取、批量并行调用工具、将宽泛搜索委派给更便宜的子代理，同时通过硬性护栏保证正确性、关键测试覆盖与代码/命令的原文不被压缩。
在 AI 编码智能体日益普及、Token 成本逐渐成为团队隐性支出的当下，token-diet 提供了一种低侵入、跨平台的通用降本方案。它不改造模型或框架，仅通过规则注入即可获得 30% 左右的账单节省，且附带可复现的基准脚本，透明可验证。对于重度使用 Claude Code、Codex 等工具的开发者和团队而言，具备极高的性价比和落地价值。

主流 AI 产品系统提示词的公开归档

MarkdownPlain TextGit
该仓库由 asgeirtj 维护，是一个专注于收集和整理主流 AI 产品「系统提示词」（System Prompts）的公开归档项目。项目将来自各大厂商官方或第三方逆向提取的系统提示原文集中存放，覆盖范围包括 Anthropic 的 Claude 家族（Fable 5、Opus 4.8、Claude Code、Claude Design），OpenAI 的 ChatGPT 5.5 Thinking、GPT 5.5 Instant 以及 Codex，Google 的 Gemini 3.5 Flash、3.1 Pro 和 Antigravity，xAI 的 Grok，还有 Cursor、GitHub Copilot、VS Code、Perplexity 等广泛使用的 AI 编程与检索工具。仓库以纯文本或 Markdown 形式呈现各产品的原始提示词，方便研究者进行横向对比、Prompt Engineering 分析、越狱与安全研究、以及 Agent 设计参考。作者声称会定期更新，跟随主流模型和产品版本迭代同步补充新版本提示词，让读者可以观察不同厂商在角色设定、工具调用、安全护栏、拒答策略、输出风格等方面的设计差异。对于开发者，它是一份可直接借鉴的 Prompt 模板库；对于研究者，它提供了研究 LLM 对齐策略与产品化落地的第一手素材；对于普通用户，则是理解「AI 助手为何这样回答」的透明窗口。
系统提示词是理解 AI 产品行为的核心工件，通常不对外公开。该仓库汇总了几乎所有主流商业 AI 助手的提示词原文，价值在于：一是为 Prompt Engineering 提供顶级团队的参考范式，二是揭示各家在安全护栏、工具调用协议、身份认知等方面的设计取舍，三是为研究模型对齐与产品化落地提供第一手素材，对开发者、安全研究者和 LLM 学术研究都具有稀缺参考价值。

SKILL.md 智能体技能规范Claude Code Plugin MarketplaceMarkdown 结构化参考文件Vercel skills CLI (npx skills)OpenAI Codex skill-installerShell 安装脚本
SAIL Skill 是 Pillar Security 推出的开源智能体技能（Agent Skill），将其 SAIL V2（Secure AI Lifecycle）安全框架封装为可被主流 AI 编码/对话代理直接加载的模块。核心内容是一份覆盖 7 个生命周期阶段、共 91 条风险的完整目录，每条风险都附带缓解措施以及对 ISO/IEC 42001、EU AI Act、OWASP LLM/Agentic、DASF、AIUC-1 等主流标准的映射，作为智能体按需读取的参考文件，确保输出引用真实 SAIL 编号与映射，而非凭空生成的控制项。技能一旦安装，在检测到 AI/Agent 安全相关意图时会自动触发，可完成智能体系统的差距评估、安全路线图与成熟度分析、合规清单生成、控制项优先级排序、跨团队对齐产物以及供应商 RFP 问卷等任务。仓库同时充当 Claude Code 插件市场，在 Claude Code 中提供 /sail、/sail:assess、/sail:roadmap、/sail:comply、/sail:rfp 五个入口命令；对 Claude.ai/Desktop/Cowork、OpenAI Codex CLI、Google Antigravity、ChatGPT、opencode、pi 等平台则通过 SKILL.md 标准或 zip 上传方式安装，覆盖面广且部署一致。项目还专门为 AI 代理准备了自安装指引，让代理可自行完成落地。
SAIL Skill 把安全咨询的专业方法论（91 风险目录 + 五大合规框架映射）标准化为智能体可加载的技能包，让任何支持 SKILL.md 的代理都能在开发流程中直接产出 CISO 级别的评估、路线图与合规清单。它示范了如何用 Skill 形态承载专业知识库、以真实条目引用替代 LLM 幻觉，并通过统一分发机制打通 Claude、Codex、ChatGPT、Antigravity 等异构生态，是 AI Agent 安全治理与 Skill 生态跨平台分发的一个标杆样例。

Claude Antigravity Agents ⭐ 74

让 Claude Code 委派任务给 Antigravity 子代理

Claude Code Skill (Markdown)Google Antigravity CLI (agy)Gemini 模型Git worktreeBash / PowerShell 安装脚本
Claude Antigravity Agents 是一个 Claude Code 技能（Skill），通过一份 SKILL.md 指令文件，将 Claude Code 从单一工作者转变为编排者（Orchestrator）。它教会 Claude Code 把重活委派给 Google Antigravity CLI（agy）子代理执行，后者是由 Gemini 等模型驱动的自主终端编码代理。工作流程为：Claude 编写自包含任务简报，以非交互方式（agy -p）在后台启动子代理，同时继续处理用户当前任务；子代理完成后，Claude 通过 git diff、类型检查和构建验证结果，再决定是否采纳。该方案带来三大价值：节省 Claude token 消耗（把全库审计、大型重构等繁重任务外包）、获得独立第二意见（Gemini 等异构模型交叉审查）、并行吞吐（多个子代理同时运行）。核心安全原则是子代理绝不写入 Claude 当前正在编辑的文件：只读任务在沙箱模式运行，写入任务在隔离的 git worktree 或独立仓库中执行，所有返回结果必须经验证方可信任。支持 Windows、macOS、Linux。
