token-diet ⭐ 568

编码代理的常驻省 Token 技能

Bash 安装脚本Claude Code SessionStart HookCodex AGENTS.mdCursor/Windsurf/Cline 规则文件Node.js 基准脚本Anthropic API
token-diet 是一款面向主流编码代理（Claude Code、Codex、Cursor、Windsurf、Cline）的常驻式 Token 节流技能。它并非按需触发的命令，而是通过安装脚本将指令注入到各代理每次会话都会加载的通道中，从而在整段会话内持续生效，包括回复、文档、测试、代码、上下文读取与工具调用等各个环节。项目核心理念是「去脂留肌」：删除寒暄与冗余叙述，回答直接给出结论；文档与注释只保留必要的「为什么」；测试聚焦关键与边缘路径，每会话不超过 10 条且绝不省略涉及金钱、鉴权、数据丢失的用例；代码遵循 YAGNI；上下文读取先 grep 再按行读取，避免整文件加载；工具调用则批量并行、够用即停。项目提供 on、lite、ultra、off 四档强度，其中 ultra 采用电报式简写但保持代码与测试精确。安装采用一行 curl 脚本自动识别代理类型，支持全局或项目级安装，实现机制上对 Claude Code 使用 SessionStart 钩子，对 Codex 使用 AGENTS.md，对 Cursor/Windsurf/Cline 使用规则文件。基准测试基于真实 Sonnet 5 运行，平均账单下降约 31%，输出体量减少 30% 至 81%，且在所有场景下保持正确性。
它把「省 Token」从零散的 Prompt 技巧沉淀为一份可安装、可开关、可基准的常驻技能，跨五大编码代理统一生效，并用真实 Sonnet 5 数据证明账单下降约 31%、正确性无损，对所有按量付费的 AI 编码用户都有直接的成本价值。

ultracodex 是一个让 Claude Code 的「一条提示衍生出一支智能体舰队」工作流脚本，无需修改即可运行在 OpenAI Codex CLI 或 OpenCode 上的桥接工具。它遵循「Fable 规划，Codex 执行，Fable 验证」的分工模式：由高端模型负责规划与审判，订阅制的执行方完成具体工作，验证后的结果回传至发起会话。项目提供 Agent Script 规范，用户通过 ES 模块编写脚本，调用 agent、parallel、pipeline、phase 等八个注入的全局函数，实现并行智能体、构建者-验证者循环、令牌预算控制。同时内置全屏 TUI 观察运行状态，支持后台脱离运行、断线续看，并通过 [route] 配置表将实现与对抗性审查分派到不同厂商。包内还附带作者技能文档与九种编排范式的示例阶梯，方便任何模型学习编写此类工作流。
在 AI 编程代理生态中，它首次实现了 Claude Code 工作流脚本在竞品 Codex CLI 上的无缝迁移，通过跨厂商模型协作显著降低高端模型使用成本，并抽象出可移植的 Agent Script 规范，为多智能体编排提供了统一的可验证语言。

SAIL Skill 是 Pillar Security 推出的开源 Agent Skill，将其 SAIL V2（Secure AI Lifecycle）安全框架封装为可跨多种 AI 助手运行的技能包。核心资产是一份覆盖 AI/Agent 全生命周期 7 个阶段、共 91 项风险的完整目录，每条风险都附带缓解措施及主流合规标准（ISO/IEC 42001、EU AI Act、OWASP LLM/Agentic、DASF、AIUC-1）的映射关系。技能以引用文件的形式在 Agent 需要时按需加载，确保输出内容基于真实的 SAIL ID 与映射，避免大模型幻觉出不存在的控制项。功能上，它可以对 LangGraph、MCP 等 AI Agent 系统做全量风险差距评估、生成 7 阶段安全路线图与成熟度画像、输出按合规框架组织的检查清单、编写供应商 RFP 问卷，以及回答类似 “SAIL 5.17 涵盖什么？” 的知识问答。在 Claude Code 中还提供 /sail、/sail:assess、/sail:roadmap、/sail:comply、/sail:rfp 五个斜杠命令作为快速入口。项目采用 SKILL.md 通用规范，兼容 Claude Code、Claude.ai/Desktop/Cowork、OpenAI Codex CLI、Google Antigravity、ChatGPT、opencode、pi 等主流 Agent 运行环境，同时作为 Claude Code 插件市场提供一键安装。
在 Agent Skill 这一新兴形态下，该项目把行业级 AI 安全框架产品化为可被任意 SKILL.md 兼容 Agent 直接调用的知识资产，解决了大模型做安全评估易幻觉、跨平台难复用两个痛点，是把合规知识工程化交付给 AI 助手的典型样板。

Claude Antigravity Agents ⭐ 99

让 Claude 委派任务给 Antigravity CLI

Claude Code SkillGoogle Antigravity CLI (agy)Shell / CLI 集成
Claude Antigravity Agents 是一个面向 Claude Code 的技能（Skill）扩展，核心定位是把编码、代码审查、分析和研究等任务从 Claude 主会话中剥离，委派给 Google Antigravity CLI（agy）驱动的子代理去执行。通过这种代理编排机制，项目实现了三个核心价值：一是节省 token，把繁重的上下文处理下沉到子代理，主会话保持精简；二是引入第二个模型进行交叉验证或补充能力，形成多模型协作；三是支持并行工作，多个子任务可同时分派给不同 agy 实例，缩短端到端耗时。该技能适合在长链路开发、代码评审和调研型任务中使用，让 Claude Code 从单一执行者转变为任务调度中枢。技术上依托 Claude Code 的 Skill 机制以及 Google Antigravity CLI 工具链，通过命令行进程调用完成子代理的分派与结果回收。
