汇聚多AI CLI观点并综合决策的技能插件

Node.jsBash ShellYAMLClaude Code SkillCodex CLIGemini CLI
Agent Council 是一个面向 Claude Code 和 Codex CLI 的多智能体协作技能插件，灵感来自 Karpathy 的 LLM Council。它通过调用用户本地已安装的 AI CLI（如 Claude Code、Codex CLI、Gemini CLI 等）来收集多方观点，再由一位可配置的主席（Chairman）综合得出最终结论。整个流程分为三个阶段：并行下发问题给各成员智能体、收集并格式化展示各家回复、由宿主智能体或指定 CLI 作为主席进行综合总结。与直接调用各厂商 API 的方案不同，该项目复用用户已订阅的 CLI，无需额外 API 费用。安装方式支持 npx 一键部署到 .claude/ 或 .codex/ 目录，也支持 Claude Code Plugin 市场安装。配置通过 council.config.yaml 声明成员、命令、颜色和 emoji，脚本层提供 start/status/results/clean/wait 等作业控制接口，便于在宿主工具 UI 中集成进度反馈。

免费 AI 网关聚合器 ⭐ 1,651 (+1651 today)

一个端点接入 268+ AI 提供商

AI GatewayMCP (Model Context Protocol)A2A (Agent-to-Agent)PWADesktop AppOpenAI Compatible API
该项目是一个基于 MIT 协议开源的免费 AI 网关，主打「Never stop coding」理念，通过统一的 API 端点聚合 268 家以上的 AI 服务提供商（其中 50 家以上完全免费），可访问 500 多个大语言模型，覆盖 Kimi、Claude、GPT、OpenAI、Gemini、GLM、DeepSeek、MiniMax 等主流厂商。项目在客户端兼容层面做得非常广泛，可无缝接入 Claude Code、Codex、Cursor、OpenCode、Cline 以及 GitHub Copilot 等主流 AI 编程工具，开发者无需修改现有工作流即可切换模型。核心能力上，网关提供配额感知的自动故障切换（Quota-aware auto-fallback），当某个上游达到额度上限或不可用时会自动路由到备用提供商，保障编码任务连续性。项目还内置了 RTK 与 Caveman 两种上下文压缩算法，宣称可节省 15% 到 95% 的 Token 消耗，显著降低调用成本。协议层面同时支持 MCP（Model Context Protocol）与 A2A（Agent-to-Agent）等新兴标准，方便构建智能体生态。交付形态上除了 API 服务外，还提供桌面客户端与 PWA 应用，覆盖多端使用场景。项目由 500 多位贡献者共同维护，社区规模较大。
在 AI 编码工具百花齐放但各家 API 割裂、成本高企的当下，该项目用一个统一端点打通 268+ 提供商与 500+ 模型，并通过免费通道+自动故障切换+Token 压缩三重手段直击开发者「额度焦虑」，还与 Claude Code、Cursor、Copilot 等主流工具原生兼容，500+ 贡献者的社区规模也说明其活跃度。对于需要在多模型之间灵活调度、又想控制成本的独立开发者与团队而言，它是一个值得关注的基础设施选项。

mentor 会话洞察技能 ⭐ 59

为 AI 编码代理生成个人使用复盘报告

PythonuvSKILL.md 规范HTML/SVGnpx skills CLI
mentor 是一款面向 AI 编码代理（Claude Code、Codex 及任何兼容 SKILL.md 格式的代理）的会话洞察技能。它在本地读取 ~/.claude 下的 Claude Code 会话记录和 ~/.codex 下的 Codex rollout 日志，聚合多次会话数据（消息数、会话数、工具调用、编辑次数、涉及文件、Token 用量、每日活跃度、工具使用比例、编程语言分布等），并生成一份自包含的 HTML 报告 report.html。报告分为八个板块：工作主题聚类、代理使用方式、亮点表现、摩擦点及对应修复建议、值得尝试的功能、可复制到 CLAUDE.md 的配置片段、新工作流建议以及趋势展望。核心价值在于摩擦分析：每个问题都对应可复用的修复方案（CLAUDE.md 配置行、hook 或习惯）。项目通过 npx skills add 一键安装，全部计算在本地进行，会话数据不上传外部，并支持 /mentor claude、/mentor codex、/mentor both 三种模式及 --days 时间窗口参数。默认输出英文，内置俄文界面。

通用 Fable 5 引擎系统提示词 ⭐ 377

精简版 Claude 泄露系统提示词，跨模型通用

MarkdownPrompt EngineeringSystem Instructions API适配 ChatGPT / Gemini / Claude
该项目是对所谓「泄露的 Claude Fable 5 / Mythos 5 系统提示词」进行的轻量化重构版本。作者认为原始泄露版本长达约 12 万字符、约 3 万 Token，其中包含大量 Anthropic 专有的 XML 工具封装、环境变量与服务器端函数调用，直接迁移到 ChatGPT、Gemini 等非 Claude 模型时会引发上下文膨胀、延迟增加甚至语义混乱。项目通过剥离 Claude 原生 `` 等 XML 包装与平台绑定逻辑，将提示词压缩到约 500 Token，同时保留原架构中的核心行为骨架，包括自主推理、多轮长程规划、自我校验循环以及一系列反聊天机器人风格约束。产物是一份可直接粘贴到 OpenAI Developer Platform、Google AI Studio、GPTs 自定义指令或 Gemini Gems 面板中的 Markdown 系统提示词。它强调四条操作哲学：反聊天机器人的连续散文输出、结构化重构输入信息、禁止思考旁白、消除结尾套话式互动，使模型呈现资深工程师式的执行风格。技术栈层面本质是纯 Prompt Engineering 项目，仅依赖 Markdown 载体和目标 LLM 的 System Instructions 接口，不含运行时代码。
围绕「泄露系统提示词」的话题近来热度很高，本项目提供了一个可验证的跨模型迁移样本，对研究不同厂商提示词架构差异、Token 预算优化以及行为约束设计有直接参考价值。需要留意的是，仓库自述基于「泄露内容」，其真实性与合规性存在争议，使用前应自行评估法律与安全风险。

AgentSmith 智能体运行框架 ⭐ 314

通用型 AI 智能体操作外壳

Bash (setup.sh / verify.sh)PowerShell 7+ (setup.ps1)Claude Code / Codex / Gemini 等智能体协议CLAUDE.md / AGENTS.md / GEMINI.md 规范文件MCP (Model Context Protocol) 服务器与插件/钩子Markdown 文档体系
AgentSmith 是一个通用、模型无关的 AI 智能体运行外壳（harness），目标是为 Claude Code、Codex、Gemini 等能读取 CLAUDE.md / AGENTS.md / GEMINI.md 的编码智能体提供一套可移植、经过实战打磨的操作系统。作者提出「智能体 = 模型 + 外壳」，认为模型只占结果的约 10%，而围绕它的规则、工具、记忆、护栏与反馈回路才是决定成败的 90%，本仓库正是把这 90% 抽象、复用化的产物。项目源自约 6 个月、数百次自主会话的真实工作沉淀，经过约五轮迭代，被拆成一个不变的通用「core」和一组可插拔的「work-type profiles」，覆盖软件开发、DevOps、市场推广、文档创作、数据处理、研究、设计、日常事务等场景。核心工作流为「安装 → 单任务 plan/implement/verify/ship → 用 verify.sh 跑真实校验 → 上下文用到约 25–30% 时自动 handoff 并沉淀经验改进外壳本身」。安装方式非常灵活：可以让智能体读取 AGENT-INSTALL.md 自助安装，也可以运行 setup.sh 向导（Windows 上对应 setup.ps1，需 PowerShell 7+），或高级用户直接传参；支持分层安装（全局 core + 项目级 profile）、安全模式（cautious/trusted）、MCP 服务器、插件包与钩子，且所有改动前会先备份。附带的 docs 目录提供哲学、首小时上手、安全模型、最佳实践等系统化文档。
在 AI 编码工具遍地开花的当下，AgentSmith 抓住了真正的杠杆点：不再纠结换哪个模型，而是把「如何让智能体稳定交付」的工程经验固化为可复用的外壳。它把「验证优先、上下文预算、早期 handoff、自主性分级、修系统而非修症状」这些通常只存在于资深使用者脑中的隐性知识，转成了一套跨模型、跨工种、可 diff、可版本化的规则组合，对希望把 Vibe Coding 升级为 AI 辅助工程的团队具有直接的落地参考价值。
