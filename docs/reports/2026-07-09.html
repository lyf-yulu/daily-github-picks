Token Diet ⭐ 566

为编码智能体降本约31%的通用技能

Agent Skill 规范Claude CodeCodexCursorWindsurfClinePrompt Engineering
Token Diet 是一款面向主流编码智能体（Claude Code、Codex、Cursor、Windsurf、Cline 等）的常驻式 Token 效率优化技能（skill）。项目定位于在不牺牲代码正确性的前提下，通过对提示词、上下文与响应结构的精细化裁剪与规范化，帮助开发者在日常 AI 编程工作流中平均降低约 31% 的 Token 消耗，从而直接压低模型调用账单。其核心思路是以一个可插拔、始终启用的规则层介入智能体的上下文构建与输出阶段：在输入侧压缩冗余上下文、去除样板化说明、精简历史对话；在输出侧引导模型采用更紧凑的表达形式、避免过度解释与重复代码块。由于 Token Diet 以“技能”而非独立工具的形态存在，它天然适配现有 Agent 生态，用户无需替换编辑器或迁移工作流，只需将该技能挂载到目标 Agent，即可获得跨模型、跨编辑器的统一节流效果。项目主打三大特性：一是通用性，一次配置覆盖多家主流编码智能体；二是无损性，声明在标准编码任务上不损失正确性；三是可度量性，用可量化的账单下降为价值提供直观证据。对于长期在大模型上进行代码生成、审查与重构的团队而言，这类“always-on”的通用降本方案能够以极低的接入成本带来持续的费用优化。README 内容较为简洁，具体实现细节需在仓库源码中进一步查阅。
在大模型编码调用成本持续走高的背景下，Token Diet 提供了一种极简接入、跨生态复用的降本方案，把“提示词工程”沉淀为可复用技能而非一次性技巧，对重度使用 AI 编码工具的个人与团队具备直接的经济价值。

SAIL Skill 是 Pillar Security 推出的 Agent Skill 插件，将其 SAIL V2（Secure AI Lifecycle，安全 AI 生命周期）框架以可加载技能的形式交付给各类 AI 编码代理和聊天助手使用。该项目内置了完整的 SAIL V2 风险目录，涵盖 AI 生命周期 7 个阶段共 91 项风险，每条风险都附带缓解措施与合规标准映射（ISO/IEC 42001、EU AI Act、OWASP LLM/Agentic、DASF、AIUC-1），Agent 按需读取参考文件，确保回答中的 SAIL ID 与映射关系均来自真实目录、不产生幻觉。技能会在用户处理 AI 或 Agent 安全议题时自动触发，可完成完整的 Gap 评估（含组件图与 91 项风险处置）、构建 7 阶段安全路线图与成熟度画像、生成多框架合规清单、优先级排序控制项、编写跨团队对齐材料，以及为安全厂商或平台厂商编写 RFP 问卷。项目同时是一个 Claude Code 插件市场，一条命令即可完成安装；对 Claude.ai/Desktop/Cowork、OpenAI Codex CLI、Google Antigravity、pi、ChatGPT、opencode 等所有兼容 SKILL.md 的 Agent 均提供了具体安装方式。在 Claude Code 中额外提供 /sail、/sail:assess、/sail:roadmap、/sail:comply、/sail:rfp 五个斜杠命令，可带智能默认值直接运行，README 还专门为 AI Agent 自助安装场景撰写了操作步骤。
在 AI Agent 安全治理仍处于早期、行业急需可执行落地框架的当下，该项目把工业级的 SAIL V2 风险目录直接封装为 Agent 可自动加载的技能，把 CISO 级的合规评审能力下沉到 IDE 与聊天窗口，同时通过 SKILL.md 标准实现了 Claude、Codex、ChatGPT、Antigravity 多平台通用，是 Agent Skill 生态与 AI 安全合规交叉领域一个值得关注的参考样板。

Claude Antigravity 代理技能 ⭐ 94

让 Claude 委托任务给 Antigravity 子代理

Claude Code SkillGoogle Antigravity CLI (agy)Shell/CLI 集成
claude-antigravity-agents 是一个为 Claude Code 设计的技能扩展，通过集成 Google Antigravity CLI（agy）来实现任务委托机制。当用户在 Claude Code 中触发编码、代码审查、代码分析或技术研究类任务时，该技能会将这些工作分派给 agy 子代理执行，从而实现三大核心价值：一是节省 Claude 主模型的 token 消耗，将重活交给外部代理处理；二是引入第二个模型（Google Antigravity 背后的模型）形成双模型协作，提高结果的多样性与可靠性；三是支持并行执行多个子任务，显著提升开发效率。该项目本质上是一个多代理协作的桥接层，让开发者能够在 Claude Code 主工作流中灵活调用外部 AI 代理完成专项工作。

认知核心技能分类体系 ⭐ 273

面向AI的通用认知技能分类框架

Cognitive-Core-Skills 是一个通用、行业中立的认知核心技能分类体系（Taxonomy），旨在为大语言模型（LLM）、小语言模型（SLM）、AI 智能体（Agent）以及世界模型（World Model）提供一套统一的能力描述与评估标准。项目将认知能力划分为八大核心域：感知（Perception）、记忆（Memory）、推理（Reasoning）、规划（Planning）、行动（Action）、验证（Verification）、学习（Learning）与治理（Governance），覆盖了从信息输入到自我治理的完整认知闭环。在此框架下，项目提供了 159 张标准化的技能卡片（Skill Cards），每张卡片可用于描述某项具体认知能力的定义、输入输出、评估指标与适用场景。项目还配套了结构化的 Schema 定义文件，用于约束技能卡片的字段规范，保证生态内数据的一致性与可机器读取性；同时内置了基准测试（Benchmarks）方案，用于对不同模型或智能体在各技能维度上的表现进行横向比较。为了保障分类体系与卡片内容的质量，仓库集成了 CI 持续集成流程，可在提交时自动校验 Schema 合法性与数据完整性。整体来看，该项目定位介于标准规范、评测框架与知识库之间，试图为异构 AI 系统建立可复用、可对齐的能力语言。
在 LLM、Agent、世界模型快速演进但能力描述各自为政的背景下，本项目尝试用工业中立的分类体系统一“AI 会做什么”的语言，把认知能力从模糊营销词变成可对齐、可评测、可 CI 校验的结构化资产，对做 Agent 评测、模型选型和能力路线规划的团队具备参考价值。
