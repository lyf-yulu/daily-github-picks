Claude Antigravity Agents 技能包 ⭐ 102

让 Claude Code 委派任务给 Gemini 子代理

Claude Code Skill (SKILL.md)Google Antigravity CLI (agy)Gemini 模型Git worktreeBash / PowerShell 安装脚本Markdown 指令文件
Claude Antigravity Agents 是一个面向 Claude Code 的技能包（Skill），核心是一份名为 antigravity-agents 的 SKILL.md 指令文件。它把 Claude Code 从单打独斗的编码助手升级为编排器（Orchestrator），能够将耗时的重活转交给 Google Antigravity CLI（agy）在后台异步执行。agy 是 Google 推出的自主终端编码代理，底层由 Gemini 等模型驱动，因此该技能相当于让 Claude 调度另一个模型作为副手协同工作。使用者只需自然语言下达任务，Claude 会自动撰写自包含任务简报、通过 agy -p 非交互式启动子代理、继续处理当前工作，待子代理完成后再收集结果并通过 git diff、类型检查和构建流程进行验证后才采信。该项目定义了严格的安全模型：只读类任务（代码审查、架构分析、依赖审计、第二意见）在沙箱中运行；写文件类任务（编码、重构）则在隔离的 git worktree 或独立仓库中执行，确保子代理永远不会与 Claude 正在编辑的文件产生冲突。项目支持 macOS、Linux 与 Windows PowerShell 环境，安装只需克隆仓库并把技能目录复制到 ~/.claude/skills/ 下，Claude Code 会在下次启动时按需加载。
它给出了一种极简却有效的多代理协作范式：不写复杂框架，仅用一份 Markdown 指令就把 Anthropic 与 Google 两家旗舰编码代理串联起来，形成成本更低、吞吐更高的跨模型工作流。对于关注 AI 编码代理编排、token 成本控制以及多模型互验证的开发者，这是一个可直接复用、思路清晰的参考实现。

Ditto ⭐ 164

挖掘AI会话日志生成你的专属agent画像

PythonClaude Code SkillsCodex Pluginnpx/skills.sh bootstrapMarkdown (you.md)
Ditto 是一个本地化的开源工具，旨在解决 AI 编程 agent「每次都像第一次认识你」的问题。它从用户在 Claude Code、Codex、Copilot CLI、OpenCode 等编程 agent 中的原始会话日志出发，挖掘出用户在实际工作中沉淀下来但从未明文写下的隐性规则：如何定义「完成」、会立刻拒绝哪类代码、如何调试、如何设计 UI、真正投入工作时的表达风格等，最终生成一份私有的 you.md 工作画像，供 agent 在每次任务开始前加载。与 CLAUDE.md 或 memory 机制不同，Ditto 不依赖用户显式告知，而是从真实历史行为中提炼「已被工作证明」的偏好，并分为工作、设计、写作等独立层，按任务加载对应人格。工具用 Python 编写，零依赖，全流程本地运行，日志不出本机，处理前会先做脱敏。运行时先输出只读的挖掘计划、成本估算与去重后的 token 数，需用户批准后才发起模型调用。挖掘完成后可通过 python ditto.py --card 生成可分享的画像卡片，包含 archetype、按会话证据排序的核心法则、覆盖统计以及一条犀利的自我洞察。安装方式包括跨 agent 的 npx skills add 引导脚本，以及原生 Codex 插件，提供 ditto:mine、ditto:work、ditto:design、ditto:write 等命名空间命令。
Ditto 提出了一个新颖的视角：与其让用户手写 prompt 或 memory，不如把 agent 已有的会话日志当成最诚实的行为数据源，反向蒸馏出个人工作风格。作者用它生成的 Reddit 帖子拿到 200 赞、10 万次浏览，形成了「用工具生成的内容验证工具本身」的闭环叙事，展示了 profile-as-context 相比传统 memory 的实际效果，对研究 agent 个性化、上下文工程和多 agent 生态的开发者都具有参考价值。

MeetingCopilot 是一款面向 Windows 与 macOS 的实时会议与面试副驾工具，主打对对方语音的实时转写和第一人称口播式答案生成，并具备防录屏采集能力。它通过 Windows 系统回环或 macOS 虚拟音频设备（如 BlackHole）直接捕获对方声音，无需以机器人身份加入会议；同时独立通道转写用户自己的麦克风。ASR 层支持四种可切换后端：本地 FunASR 流式、本地 Whisper turbo（DirectML）、阿里云 fun-asr-realtime 以及 MiMo 分段模式，中英文自动切换。答案由用户自带 Key 的 OpenAI 兼容 LLM（推荐 DeepSeek）生成，采用结论先行、STAR 结构或技术三段式，并结合本地解析的简历与 JD，避免虚构经历。此外还提供滚动面试备忘、前缀缓存预热、区域截图视觉问答、全局隐藏热键与中英双语 UI 等能力。
