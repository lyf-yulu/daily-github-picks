token-diet ⭐ 585

编码 Agent 通用的 Token 瘦身技能

Bash 安装脚本Node.js 基准工具Anthropic APIClaude Code HooksAGENTS.md / Cursor Rules / Windsurf Rules / Cline Rules
token-diet 是一个面向编码类 AI Agent（Claude Code、Codex、Cursor、Windsurf、Cline 等）的常驻式 Token 节省技能。它通过在会话初始化阶段注入一份指令，让 Agent 在整个会话周期内自动收敛输出：回复直接给结论、避免寒暄和收尾；文档、注释、交接说明只保留必要信息；测试聚焦关键与边界路径且单次不超过 10 条；代码遵循 YAGNI 原则，不生成死代码；上下文读取先 grep 再定点读行、批量合并、避免重读；工具调用并行且在拿到足够信息后立即停止；对宽泛搜索委派给更廉价的子 Agent。项目提供 on、lite、ultra、off 四档强度，其中 ultra 采用电报式风格进一步压缩对话，但保证代码、命令、错误原文等关键内容原样保留，不牺牲正确性。安装通过一行 curl 脚本完成，自动识别目标 Agent，可全局或仅当前仓库生效，并支持运行时通过 /token-diet 指令切换等级。作者在 Sonnet 5 上给出了真实基准：输出平均减少约 53%，账单平均下降约 31%，输出密集型场景最高可省 54%。
在 AI 编码 Agent 大规模落地的当下，Token 成本已是实际生产痛点。token-diet 用极轻量方式（一份 prompt 指令 + 各 Agent 的常驻注入通道）就跨平台解决了成本问题，并给出可复现的基准数据，避免了对模型或框架的侵入式改造，具有很强的普适性和即时可用性。

loopkit ⭐ 312

33 个即插即用的编码 Agent 技能包

ShellMarkdown (SKILL.md)Claude CodeMCPJSONCursor / Codex / Gemini CLI
loopkit 是一个面向编码类 AI Agent 的开源技能库，由 33 个经过实战打磨的 skill 文件以及一套极简的 .claude 运行骨架组成，目标是让 Claude Code、Cursor、Codex、Gemini CLI 等任意读取 SKILL.md 的编码 Agent 都能输出更干净、更少臆测的代码。项目核心理念是：原生 Agent 在缺少约束时会靠猜，进而虚构 API、写出永远通过的假测试或做出改名式的伪修复；而每个 skill 都是一段紧凑、可复用的指令集，只在触发条件命中时才被加载，让 Agent 在特定场景下专业化，同时避免每次提示都为所有技能付出 token 成本。33 个技能被划分为 9 条轨道，覆盖 agent/LLM 工程（上下文预算、spec 优先、工具克制、子 Agent 扇出）、系统化调试、安全审查（OWASP、鉴权、输入校验、密钥扫描、依赖审计）、前端设计系统与可访问性、TDD 与 flaky 测试、重构、文档与决策记录、SQL 与数据迁移、Git 提交与 PR 规范，以及一个对抗式 verifier 用于识破 Agent 常见的 11 种偷懒手法。安装脚本会在当前目录写入 .claude/CLAUDE.md 标准上下文、settings.json 权限白名单、verifier 子 Agent、全部技能文件、.mcp.json、MEMORY.md 跨会话记忆索引以及 Plan→Act→Verify 循环运行器 run.sh，构成完整的 Loop and Harness 骨架。MIT 协议，鼓励 fork 后按团队实际情况调整。
在多数人还在给 Agent 写巨型 system prompt 时，loopkit 提出用可组合的技能文件 + 触发式加载来分摊上下文，并配套一个仅 7 文件 5 步的 Loop and Harness 参考实现。它把 Anthropic 关于 Building Effective Agents 的方法论落成了可直接 curl 到项目里的工程模板，尤其是 adversarial-verify 这类专门识破 Agent 假装完成的技能，对追求可交付质量的团队具有较高参考价值。

agent-runtime ⭐ 141

厂商中立的独立 Agent 运行时内核

Python 3.11uv (包管理)pytestOpenAI Chat Completions APIAnthropic APIApache License 2.0
agent-runtime 是一个独立、可复用的 Agent 运行时核心库，专注于沉淀「请求 → 工具调用 → 循环 → 收尾」这一稳定的 Agent 主循环逻辑，与具体产品应用（如会话、存储、沙箱、品牌策略等）彻底解耦。项目通过一套厂商中立的数据模型（Message、TextPart、ImagePart、ToolCallPart、LLMRequest、LLMResponse 等）来描述会话与模型调用，避免直接依赖 OpenAI 或 Anthropic 的原始字典结构，并内置了对应的 wire converter 在中立模型与各厂商线上格式之间转换。使用方只需传入 LLMConfig，即可自动构建对应的 HTTP 客户端；同时通过 ModelClient、ToolDispatcher、SystemPromptProvider、CacheStrategy 四大注入协议，允许产品层灵活替换工具分发、系统提示词组装、缓存策略等实现。运行时还提供了协作模式（CollaborationMode，只提供机制不硬编码策略）、生命周期 Hooks、迭代预算、Token 估算、上下文压缩（SummarizingCompactor）、流式回调、模型重试、工具错误容错、人机交互暂停（WaitingForUserInput）与中断处理等生产级能力。项目坚持「内核只做机制、产品层做策略」的边界划分，TurnResult 只承载消息与元数据，不与持久化耦合，测试也完全基于 fake 与 monkeypatch，不依赖网络与数据库。
在 Agent 框架普遍与业务、存储、Web 服务器深度耦合的当下，agent-runtime 提供了一个罕见的「纯内核」样板：它把 Agent 循环、协作模式、Hooks、上下文压缩等稳定逻辑抽出来，用中立类型作为边界，产品层只需组合协议即可接入自己的工具、记忆、沙箱与持久化。这种「机制与策略分离」的架构对想自研 Agent 平台、又不愿被 LangChain/LlamaIndex 等重型框架绑定的团队具有很高的参考价值，代码规模可控、测试无外部依赖，也非常适合作为学习 Agent 内核实现的样例。
