RealmForge 是一个开源的生成式智能体群（Agent Swarm）编排平台，定位于构建具备持久状态与叙事连贯性的模拟世界。项目面向世界观构建者、游戏设计师以及研究涌现社会行为的学者，试图弥合大语言模型的创造性与持久化模拟所需结构之间的鸿沟。其核心是结构化世界模型（SWM）：一张由智能体、地点、物品与概念组成的有向图，每个实体带有属性、库存与叙事历史，智能体基于共享的图状现实进行读写，避免了对话之间遗忘世界的问题。系统架构分为三层：Conductor 编排层负责 Tick 队列与 Epoch 循环调度，防止图写入竞态并暴露 REST API；Cortex Pool 作为 LLM 接口层，聚合本地 llama.cpp 与云端 API，负责提示词组装、路由与压缩；Index 持久化层则由世界图、时间序列的 Chronicle 事件日志与向量记忆库构成，支撑 RAG 式的智能体回忆。特色能力包括感知半径造成的信息不对称、Persona Script 加 State Vector 的多身份编排、以欲望竞价驱动的叙事市场（Narrative Market）、多语言角色互动，以及以 Epoch 为单位的非线性时间推进。前端提供 TUI 反应式控制台与 Webhook 子系统，可对接 Web 面板、Discord 机器人或直播叠层。
该仓库在描述上试图切入 2026 年 LLM 世界建模与智能体群模拟的前沿方向，理念上融合了 AgentOS、生成式智能体与知识图谱三条路径，narrative market 与感知半径的设计对 NPC 模拟与社会仿真研究有一定参考价值。需要提示的是：仓库名 mirage-cortex 与 README 中的 RealmForge 品牌不一致，且 README 中仅提供指向外部 GitHub Pages 的下载按钮，未见实际源码与提交历史披露，建议在实际使用前核实代码可用性与来源可信度。

OpenTag ⭐ 381

在 Slack/GitHub 中 @召唤本地编码智能体

Node.js 20+TypeScriptpnpmnpm (@opentag scope)Slack Socket ModeGitHub WebhooksLark/Feishu APICodex CLIClaude Code CLI
OpenTag 是一个开源工具，让团队可以在日常使用的协作平台（Slack、GitHub、Lark/飞书）中通过 @提及的方式召唤本地编码智能体完成任务。它监听指定平台的消息事件，将带标签的请求路由到本机运行的 Codex 或 Claude Code，然后把执行结果回传到同一条会话线程中。整个 CLI 流程采用本地优先架构：没有 OpenTag 云服务，平台凭证以私有文件权限存储在用户机器上，编码智能体直接在本地代码检出目录上运行，平台 API 仅接收确认、回复及用户批准的操作所需的最小消息。核心设计亮点是「源线程操作回执」机制：当智能体建议执行系统级变更时，OpenTag 会在原始会话线程中渲染紧凑回执，明示将要发生的变更、是否可执行，以及当前的安全决策，只有在调度器确认已配置适配器可执行时才显示 Apply 按钮。项目提供 opentag setup、start、status、doctor 等命令，通过四个问题（监听平台、编码智能体、本地项目、平台凭证）完成引导式配置。运行时由本地 dispatcher、runner 和平台 listener 三部分组成，配置保存在 ~/.config/opentag/config.json，隔离 worktree 存放于 ~/.local/state/opentag。已发布 v0.3.0，通过 npm 以 @opentag 作用域发布 CLI、local-runtime、core 三个主要包，要求 Node.js 20+。
把「@某人」这一最自然的协作动作直接映射到编码智能体，弥合了团队沟通与 AI 编码之间的操作鸿沟。相比常见的云端 AI 助手，它坚持本地执行 + 线程内审批的模式，兼顾了企业代码隐私与协作透明度，特别适合已经在用 Codex 或 Claude Code 的团队将其扩展为面向全队的可召唤能力。
