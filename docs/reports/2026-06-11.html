BrandDocs 品牌文档生成器 ⭐ 144

AI 智能体专用的品牌文档生成技能包

Agent Skills（Claude Code / Codex）Microsoft Word (.docx)Microsoft PowerPoint (.pptx)Microsoft Excel (.xlsx)Office Open XML
BrandDocs 是一套面向 AI 智能体的技能（agent skills）集合，专注于解决企业在使用通用 AI 文档生成工具时面临的品牌一致性难题。其核心定位是：让 AI 智能体学习用户已有的 Word、PowerPoint 和 Excel 模板，并基于这些模板生成符合品牌规范的新文档。与市面上常见的通用 AI 文档生成器不同，BrandDocs 通过结构化的方式从根本上保证生成结果保留原有的品牌元素、文档结构、样式定义以及 Excel 公式，避免了 AI 生成文档时常见的格式错乱、品牌偏离、公式失效等问题。该项目专为 Claude Code、Codex 等兼容的 AI 编程智能体设计，作为可加载的技能模块被调用。在功能上，BrandDocs 覆盖 Office 三件套的主要文档类型，包括用于撰写报告与提案的 Word、用于商务汇报的 PowerPoint 以及用于财务建模与数据分析的 Excel。其技术路径强调“by construction”（构造式保真），即不通过事后修复而是在生成阶段就锁定品牌规范。这一思路对于咨询公司、品牌团队、企业内部办公自动化等强依赖模板一致性的场景尤为契合。
在 AI 文档生成赛道日益拥挤的当下，BrandDocs 切入了一个被普遍忽视但企业刚需的痛点——品牌与模板一致性。它不与 ChatGPT、Gemini 这类通用生成器正面竞争，而是作为 Agent 生态中的“技能层”嵌入工作流，体现了 AI 智能体技能化、模块化的最新趋势。对于希望将 AI 能力落地到合规、品牌敏感场景的团队，这类“保真生成”方案可能比从零生成更具实用价值。

SkillRevise ⭐ 50

基于轨迹的LLM智能体技能修订框架

PythonLLM APIAgent Framework
SkillRevise 是一个面向 LLM 驱动智能体（Agent）的技能修订框架，专注于解决大模型自动生成的 Agent Skills 在实际执行中可能存在的缺陷与偏差问题。项目提出 trace-conditioned revision（轨迹条件修订）方法，即通过收集智能体执行技能时产生的运行轨迹（trace），将这些真实执行信号作为条件输入，反馈给 LLM 对原始技能脚本进行有针对性的迭代修订，从而提升技能的鲁棒性、正确率与可复用性。该仓库为论文配套的官方开源实现，预期包含技能生成、轨迹采集、修订流水线、评测脚本等核心模块。由于 README 内容当前仅为简介，技术栈细节需以实际代码为准，但典型实现通常基于 Python 与主流 LLM API。该项目对希望提升 Agent 自我改进能力、构建可靠工具调用与技能库的研究者具有参考价值。

Hephaestus 是 Agentlas 平台背后的核心引擎，定位为无代码 AI Agent 构建器。它的核心能力是把用户输入的一句自然语言需求，直接转化为可运行的单个智能体或多智能体协作团队，免去了手工编排 Prompt、定义角色、配置工作流的繁琐过程。项目将自身定位为 CrewAI、LangGraph、AutoGen 等主流多智能体框架的无代码替代方案，但在落地形态上做了差异化：不绑定单一运行时，而是直接运行于 Claude Code、Codex、Cursor 等开发者已经在使用的 AI 编码环境之上，借助这些宿主提供的模型调用、工具执行与上下文管理能力。这种设计让用户无需自建后端或部署额外服务，即可在熟悉的 IDE/CLI 中拥有一套可扩展的 Agent 编排体系，适合希望快速验证多智能体协作场景、又不想陷入框架学习成本的开发者与团队。

豹语设计 (Baoyu Design) ⭐ 736

本地运行的 Claude Design 技能包

Claude Agent SkillsMarkdown 提示词工程HTML/CSS/JavaScript (单文件输出)Cursor / Claude Code 等 Agent 宿主
豹语设计 (baoyu-design) 是一个将 Claude Design 能力封装为 Agent Skill 的开源项目，让用户无需依赖 claude.ai/design 在线服务，即可在 Cursor、Claude Code 等本地 AI 编程环境中调用 Claude 的设计生成能力。项目核心目标是把 Anthropic 官方的设计技能下沉到本地工作流，使开发者和设计师可以直接在自己熟悉的编辑器或终端中，通过自然语言指令产出高质量的 UI 界面与原型成果。其产出形式统一为自包含 (self-contained) 的单文件 HTML，内联样式与脚本，便于预览、分享、版本管理和二次迭代。支持的产物类型涵盖 UI 模型 (mockup)、可交互原型 (prototype)、演示幻灯片 (deck) 以及线框图 (wireframe) 四大场景，覆盖产品设计从概念到展示的完整链路。该 Skill 遵循 Claude Agent Skills 的规范，通过 Markdown 描述能力边界与提示词模板，运行时由宿主 Agent (如 Claude Code) 加载并调用。项目特别推荐搭配 Claude Opus 4.8 模型使用，以获得最佳的视觉表现与交互逻辑生成效果，对低算力模型也保持兼容。整体上它是一个轻量但实用的工程化封装，把云端封闭体验转化为可本地化、可定制、可嵌入团队工具链的标准化技能模块。
随着 Anthropic 力推 Agent Skills 生态，将官方在线产品能力反向移植为本地 Skill 是一个值得关注的趋势。该项目以最小成本打通了 Claude Design 在 Cursor、Claude Code 等本地环境的使用路径，免去了在 claude.ai/design 与编程工具间反复切换的成本，且产物为可直接交付的单文件 HTML，对设计驱动的快速原型、产品评审、AI 辅助前端开发都有直接价值，是观察 Skills 生态如何替代封闭 SaaS 体验的代表性样本。
