Ditto ⭐ 179

从AI编程日志挖掘个人工作画像

Python（零依赖）Model Context Protocol (MCP over stdio)Claude Code Skills / Codex Plugin 生态npx skills 分发
Ditto 是一个开源的本地化工具，用于从开发者与 AI 编程代理（Claude Code、Codex、Copilot CLI、OpenCode、Google Antigravity 等）的历史会话日志中，挖掘出使用者真实的工作习惯与偏好，并生成一份私有的 you.md 工作画像。它的核心理念是「不是记忆，而是证据」：记忆是你明确告诉模型的内容，而 Ditto 挖掘的是你的实际工作过程所证明的事实——你如何定义『完成』、你会立即拒绝什么样的代码、什么时候要求验证、什么样的代理行为会让你终止任务、以及你在真正工作时的表达方式。工具会读取原始会话日志，剥离工具输出、错误粘贴和助手回复，只保留用户自己的语言，然后分片交给代理分析归纳，最终合并出分层画像（work、design、writing 三层，按任务加载对应部分）。使用者可通过 python ditto.py --card 生成可分享的画像卡片，展示原型定位、按证据条数排序的规则、覆盖统计和一句核心洞见。分发上，Ditto 提供跨代理 bootstrap（npx skills add）、Claude Code 与 Codex 原生插件（含 mine/work/design/write 四个技能），以及一个基于 stdio 的 MCP 服务器，暴露 load_ditto_profile 工具，供 Claude Desktop、Cursor 等任意 MCP 客户端在任务前加载画像。项目采用 Python 实现，零依赖，MIT 协议，所有日志处理均在本地完成且会先做敏感信息脱敏。
Ditto 提出了一个反直觉但很有说服力的视角：AI 代理最缺的不是更多的 prompt 或 memory，而是对使用者真实工作方式的证据化理解。它把会话日志作为一种被长期忽视的一手数据源，用『挖掘 + 分层 + 任务前加载』的范式，把个人工作习惯转成可复用的代理配置。作者用 Ditto 本身写的 Reddit 帖子拿到 200 赞、102K 浏览，形成了产品即案例的自证闭环。加之零依赖、本地运行、多代理兼容（含 MCP），对关注 AI 编程工作流个性化与隐私的开发者具有较强的参考价值。

MeetingCopilot 是一款面向 Windows 与 Apple Silicon macOS 的实时会议 / 面试副驾工具，主打「隐身式」提词器体验。它直接捕获对方声音（Windows 使用系统 loopback，macOS 借助 BlackHole 等虚拟音频设备），无需任何会议机器人参与，并通过独立的麦克风通道分别转写自己的发言。ASR 后端可在四种方案间切换：本地 FunASR 流式（默认，Python sidecar 由主程序托管）、本地 Whisper turbo（DirectML GPU 离线兜底）、阿里云 fun-asr-realtime 逐字流式、以及 MiMo 分段云端识别，中英文可在同一会话中自动切换。答题侧采用 BYOK 的 OpenAI 兼容 LLM（推荐 DeepSeek），以第一人称、可直接朗读的口播文风生成回答：结论先行、行为题走 STAR、技术题按思路 → 要点 → 复杂度组织，并严格基于用户导入的简历 / JD（本地解析 md/txt/docx/pdf，不上传）。项目还提供滚动面试备忘、prefix-cache 预热以降低首字延迟、区域截图交给视觉模型问答、全局隐藏热键与 Windows 端窗口防录屏、中英文界面即时切换、多会话隔离、暗色 / 亮色主题以及内联翻译等能力，构成了一套完整的实战面试辅助系统。
它把「实时 ASR + LLM 提词器」这条被反复尝试的赛道做到了工程闭环：本地流式 FunASR 让隐私与成本可控，DeepSeek prefix-cache 预热和滚动备忘把长会话下的首字延迟与上下文一致性同时压住，再配合 Windows 端窗口级防录屏与全局隐藏热键，形成了「对端能听清、屏幕共享看不见、答案可朗读」的差异化体验，对研究实时语音 Agent、面试辅助与桌面隐身应用的开发者都具备参考价值。

EasyLastSkill 是一款面向 Codex/Agent-Skills 生态的德语调研技能，由 Dr. René Bäder 开发并以 GPL v3 协议开源。它的核心定位是围绕某个主题快速抓取近段时间（默认 30 天）内的公开信号，并生成一份带来源引用、相关性评分和不确定性说明的紧凑德语报告。用户可以通过命令行传入关键词，指定回溯天数、输出格式（Markdown 或 HTML）以及保存路径，也可以启用「A vs B」比较模式对两个主题做横向对照。项目的 0.2 版本在合成质量上做了明显强化：报告结构划分为短结论、新信号、背景归纳与不确定性四段，并新增了针对人物、公司、产品、仓库和研究等不同类别的主题画像。数据来源覆盖 Google News RSS、Hacker News Algolia API、GitHub Search API、Reddit RSS、arXiv API、YouTube 公共 RSS 搜索以及可选的 Bluesky 搜索接口，全部为无需密钥的公开渠道。相关性打分综合考虑时效性、来源权重、社区互动和主题匹配度，遇到网络受限或结果不足时也会在输出中显式提示。项目还提供 easy-last-skill.toml 配置文件以及 build_skill_bundle.py 打包脚本，便于以 .skill 包形式集成到 Agent Skill 工具链中。
在 Agent Skill 生态中，多数示例仍以英文为主，EasyLastSkill 以德语原生输出并聚焦「近期动态调研」这一具体工作流，展示了 Skill 如何从通用 Chat 能力收敛为面向特定语种和场景的可复用工具。其无密钥多源聚合与显式的相关性评分、不确定性提示，也为构建可解释的 AI 调研管道提供了参考范式。

OpenFinClaw CLI（@openfinclaw/cli）是一款面向量化投研场景的命令行工具，通过 MCP（Model Context Protocol）协议将完整的量化工作流接入 Claude Code、Cursor、VS Code、Windsurf、Codex 等 20 多种 AI Agent 平台。用户只需一句自然语言 prompt，即可在终端或 IDE 内串起「研究 → 策略生成 → 回测 → 模拟交易 → 发布上榜」的端到端流程。工具内置 DeepAgent 分析引擎，提供 60 多种分析技能，涵盖技术面、基本面、情绪面、风险、择时和多因子等维度，覆盖美股、A 股、港股、加密货币和外汇五大市场。CLI 采用 npx 一键安装模式，install 向导会自动检测本地 AI 客户端并写入对应的 MCP 配置文件，同时将 fch_ API Key 以 chmod 600 权限持久化到 ~/.openfinclaw/config.json，并在 ~/.claude/skills/ 下注册 SKILL.md 以便 Claude Code 通过关键词自动触发。命令体系分为 DeepAgent、Strategy、Raw API 和系统命令四组，支持流式输出的 +verb 形式和面向脚本调用的原子三元组 research_submit/poll/finalize。社区侧提供类 Hugging Face 的策略交易所，用户可 leaderboard 浏览、fork 复制、validate 校验后 publish 到服务端进行统一回测排名，保证成绩真实可比。
它把「AI Agent + 量化投研」这两个高热赛道用 MCP 优雅缝合：不再需要为每个 IDE 或 Agent 单独适配，一次安装即可让 Claude Code、Cursor 等 20+ 客户端具备完整的多市场量化能力，并配套社区策略市场与服务端统一回测机制，为量化策略共享提供了「Hugging Face 式」的可信基础设施，具备较强的生态卡位价值。
