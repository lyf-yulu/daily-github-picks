TokDiet ⭐ 69

AI编码代理的本地计费与降本代理

Python流式 HTTP 反向代理Anthropic / OpenAI / Gemini / MiniMax API本地 Web Dashboard
TokDiet 是一个面向 AI 编码代理（Claude Code、Cursor、Codex）的本地流式反向代理，位于代理与各大模型 API（Anthropic、OpenAI、Gemini、MiniMax）之间。它会实时拦截并转发请求与响应流，对每一次调用进行 token 用量与美元成本的精细计量，类似 ccusage 风格的度量，并通过本地实时仪表盘可视化展示。其核心增值在于上下文压缩：当上下文体积膨胀、推高按 token 计费的 API 支出时，TokDiet 会在保留语义完整性的前提下精简冗余内容，从而显著降低成本。同时它内置 shadow-eval 影子评估机制，对压缩前后的输出质量进行对比验证，证明降本不会牺牲生成质量。整体方案部署在本地，对开发者透明，无需更换客户端即可接入，适合希望追踪、审计并优化大模型调用成本的个人与团队。

Harness-Forge ⭐ 67

让 Claude Code 自我进化脚手架

Claude CodeClaude Code SkillsPareto 多目标优化Prompt EngineeringRAG / 检索增强
Harness-Forge 是 Meta-Harness（Lee et al. 2026）的原生复现实现，将 Claude Code 转化为一个能够自我演化的元脚手架（Meta-Harness）。项目的核心思想是：在保持底层模型固定不变的前提下，通过迭代演化模型周边的脚手架组件——包括记忆（memory）、检索（retrieval）、上下文（context）和提示词（prompts），来持续提升整体智能体的表现。其工作机制基于一个原生的 propose→score→Pareto 三阶段循环：首先生成候选脚手架方案，然后对候选方案进行多维度评分，最后通过 Pareto 前沿筛选保留在多目标上均表现优秀的方案。这种方式避免了对模型本身的微调成本，转而通过优化模型外围的工程化组件来获得性能增益，是一种轻量但有效的智能体自进化范式。项目以 Claude Code Skill 的形式交付，方便集成到现有 Claude Code 工作流中。
在不微调模型的前提下，通过 Pareto 进化脚手架组件实现智能体自我提升，是一种新颖的轻量级自进化范式。
