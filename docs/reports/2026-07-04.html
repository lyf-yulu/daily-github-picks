RealmForge 是一个开源的生成式智能体群编排平台，旨在为持久化、叙事连贯的模拟世界提供底层框架。项目的核心创新是「结构化世界模型」（Structured World Model, SWM）——以有向图形式存储智能体、地点、物品与概念等实体，让 LLM 驱动的角色在共享现实中读写状态，从根本上解决传统对话式智能体「会话即遗忘」的问题。系统采用三层架构：Conductor 编排层负责 Tick 队列调度与 Epoch 时钟推进，处理事件与冲突优先级；Cortex Pool 管理 LLM 请求路由、Prompt 组装与压缩，支持 llama.cpp 本地模型与云端 API 混合部署；Index 持久层则保存世界图、事件编年史与向量记忆库，实现 RAG 式的真实回忆。功能层面引入感知半径、关系衰减、Persona 脚本、多语言角色互动、叙事市场（Narrative Market）以及随机环境事件等机制，可催生涌现式剧情与复杂社会经济系统。项目附带 TUI 反应式控制台实时可视化世界图，并通过 Webhook 广播 JSON 事件，便于对接 Web 前端、Twitch 直播覆盖层或 Discord 机器人，面向世界观创作者、游戏 NPC 设计师与 LLM 社会行为研究者。
RealmForge 的价值在于把「LLM 智能体」从单轮对话推向持久化模拟：通过结构化世界图 + Epoch 时钟 + 叙事市场三件套，形成了一个可复用的『虚拟社会操作系统』雏形。对世界观创作、深度 NPC 原型以及多智能体社会行为研究都具有参考意义，尤其是 Prompt 压缩与向量记忆的组合思路，回应了长上下文成本高、角色易失忆两大痛点。需要留意的是，README 目前偏概念叙述，缺少代码样例与依赖清单，读者可结合仓库源码进一步核验其成熟度。

让 AI 编程助手把踩坑经验沉淀成可复用技能

Agent Skills 开源标准Markdown（SKILL.md / AGENTS.md / .mdc）Claude Code Plugin / MarketplaceCursor Rulesnpx skills CLIShell 脚本
self-learning-skills 是一个面向 AI 编程助手（Claude Code、Cursor、Codex、Zed、Aider、Gemini CLI 等）的自我进化『元技能』。它解决了一个长期痛点：每次会话中辛苦调试得到的『黄金路径』——比如如何连接生产数据库、凭据在哪里、部署命令是什么、如何在线验证等——会随会话结束而蒸发，下次又要从零重新摸索。该项目并不直接完成开发工作，而是教会 Agent 识别『刚刚赚到一段可复用经验』的时刻，并将其自动持久化到工具会自动加载的位置。核心循环由三步组成：识别时机（多次尝试才成功、非显然命令、用户显式说 remember this 等）→ 无需提示地自主抓取（选定作用域与命名，同时记录『什么行不通』）→ 下次会话自动加载复用。项目为不同工具提供了对应适配：Claude Code 落地为 skills//SKILL.md，Cursor 落地为 .cursor/rules/learned/.mdc，其他 Agent 则追加到 AGENTS.md。它还引入了分诊机制（技能 / 备忘 / 跳过）与晋升规则（必须同时具备通过验证、命名失败模式、排除的死路），避免把未验证的猜测固化。安全上明确不写入任何秘钥值，只记录秘钥所在位置。
这是一个把『AI 会遗忘』这个日常痛点转化为工程化解法的巧妙尝试。它没有堆砌复杂框架，而是用一个跨工具通用的 Markdown 约定 + 分诊 + 晋升规则，把 Agent 的一次性经验沉淀为长期资产，同时用『只记位置不记值』守住秘钥安全底线。对于重度使用 AI 编码助手的团队，这种可共享、可 Git 管理的『黄金路径库』有明显的复利价值，且思路可直接迁移到自研 Agent 生态中。

reverse-flow-skill 是一个面向 AI Agent / Codex 的本地 CTF 逆向工程流程技能包，通过启动词『真心为你』进入逆向模式，为智能体在本地沙盒、CTF、crackme、wargame 及授权训练靶场中执行逆向任务提供稳定的工作流。项目将逆向过程拆解为『分析 → 报告 → 逆向 → 深度逆向 → 漏洞研判 → 用户选择下一步』六阶段流水线，覆盖样本分诊、工具链审计、静态分析、深度逆向、漏洞研判到报告输出的完整闭环。技能采用『英文内核提示词 + 中文用户交互』的双语设计：内部执行规则、工具选择与流程控制使用英文以提升模型稳定性，而启动词、报告结构与下一步菜单则保持中文，兼顾模型可靠性与用户体验。项目内置口语化需求归一化能力，可将『解锁 XX』『去除 XX』『绕过反调试』『拿 flag』等模糊表达自动映射到 feature gate 定位、check routine 分析、anti-debug 处理等具体动作，并强调在副本上给出 patch 或调试方案。仓库通过 SKILL.md、agents/openai.yaml 等文件描述技能定义，并提供 create_case、triage_artifact、tool_audit、report_from_triage 等 Python 脚本用于案例创建、样本分诊、工具审计与报告生成，安装方式为将技能目录复制到 Codex 的 skills 目录下，遵循 MIT 协议开源。
该项目把『AI Agent + 逆向工程』这一新兴场景固化成可复用的技能包，用工作流约束替代自由发挥，显著降低智能体在 CTF/crackme 场景下的输出漂移；同时其双语提示词架构与口语化归一化表，对想要构建垂直领域 Agent Skill 的开发者具有很好的参考价值。

MindMirror ⭐ 151

MindMirror（仓库名 copilot-cache-scout）自称是一款针对多轮 AI 对话的诊断工具包，用于衡量三种缓存策略下的真实 Token 成本：朴素记忆保留（Naive Memory Retention）、提示级缓存（Prompt-Level Caching），以及项目主打的“图书馆员摘要模式”（Librarian-Pattern Digest）。该模式借鉴人类图书馆员维护主题索引的方式，动态维护对话状态的语义压缩图谱，宣称在多智能体代码评审场景下相较提示缓存能降低 27%–41% 的 Token 消耗。工具原本为在 GitHub Copilot 代理上运行 Claude Opus 4.6 的评审场景设计，现已扩展为通用对话式 AI 分析框架，支持 3–7 个评审 Agent、2–50 轮对话深度的压力测试，并输出 JSON 报告、CSV 时间线以及基于 D3.js 的响应式可视化仪表盘。附加功能包括多语言（英日中德法）输出、暗色/亮色主题、对话回放动画、PDF/PNG 导出等。需要注意的是，README 中提到的 “Claude Opus 4.6” 目前并不存在，多处“Download”按钮指向外部 GitHub Pages 而非源码，仓库名与项目名不一致，宣传口吻偏营销化，实际代码与所述功能是否匹配需读者自行核实。
