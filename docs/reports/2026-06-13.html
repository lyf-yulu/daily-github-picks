architect-loop ⭐ 118

Claude架构师与Codex构建者协同闭环

Claude Code SkillsOpenAI Codex CLI (>=0.133)Bash / PowerShell 安装脚本Git worktreePython (tests/validate_skills.py)Markdown 规格与门控文件
architect-loop 是一个跨厂商 AI 代理协作框架，以 Claude Code Skill 形式实现「架构师-构建者」分工循环。项目让 Claude Fable 担任规划与评审的架构师角色，由 GPT-5.5 Codex 担任实施与研究的构建者，仓库本身作为唯一的共享记忆。它无需 API Key，直接复用用户已有的 Claude 付费订阅与 ChatGPT 订阅额度。 核心提供两条命令：`/architect` 负责构建闭环，Fable 在每个工作块先写规格与验收门（gates）并提交至 `docs/gates/`，然后将任务切分为 1-4 个不重叠的车道（lane），每条车道在独立 git worktree 中由全新的 `codex exec` 进程并行执行；构建者无提交权限，最终由 Fable 在新会话中亲自跑门控命令、审阅 diff，再合并通过的车道。`/architect-research` 则用于决策前的研究，遵循 scout-first 模式：先由廉价 Codex 侦察员绘制主题图谱，Fable 据此动态设计 3-6 条研究车道，并行 Codex 研究员在硬预算下采集带 URL、日期、引文与置信度的证据，最后由 Fable 进行对抗性证伪与撰写。 所有设计都附带论文与工程实践引用（见 DESIGN.md），强调外部冻结门、worktree 隔离、防记忆腐烂的简短 HANDOFF 索引以及活性检查与停滞分诊等运维细节。
它把近年来关于 AI 代理协作的多项研究结论（弱规划者比弱执行者更致命、代理会刷可见测试、记忆文件会腐烂、生产级 deep-research 用规划者动态分解而非固定车道）凝练成一套可直接安装的工作流，并以跨厂商方式同时榨取 Claude 与 Codex 两个订阅的能力上限——这种用 Skill 文件而非新框架来落地多代理工程实践的做法，对正在探索 agent loop 的开发者很有参考价值。

SkillRevise ⭐ 50

基于执行轨迹修订LLM代理技能

PythonpytestOpenAI APIALFWorldCLI
SkillRevise 是论文《SkillRevise: Improving LLM-Authored Agent Skills via Trace-Conditioned Skill Revision》的官方代码发布。项目将代理技能视为可执行的产物，针对冷启动场景下 LLM 生成技能不够可靠的问题，提出轨迹条件化的技能修订框架：先由 LLM 撰写初始技能并执行任务，再通过验证器反馈进行失败诊断，从可复用的原则记忆库中检索修复经验，结合执行锚点修订技能，重新执行候选并通过效用门控选出最优版本。框架由任务级 Diagnosis、可复用 Principle Memory 和带锚点的 Revision Operator 三部分构成。仓库提供统一的 SkillsBench 风格基准接口，内置 SkillsBench、SkillLearnBench-Random、SWE-Skills-Bench-Hard 三套基准包。在 GPT-5.5 主实验中，三轮修订预算下 SkillRevise 在三个基准上均显著超越无技能基线和一次性 Skill-Creator，并展现出跨模型迁移能力。技术栈以 Python 为主，提供 CLI、命令式 LLM 客户端和 pytest 测试。

baoyu-design 把 claude.ai/design 背后的设计引擎打包成一个可移植的 Agent Skill，让用户在 Cursor、Claude Code、Claude Desktop、Codex 等任意支持文件操作的本地编码 Agent 中直接生成精美的 UI 稿、可交互原型、线框图、落地页、仪表盘、移动应用与幻灯片，全部以自包含 HTML 形式输出到 designs// 目录，无需访问官网、无需额外订阅、也无需上传素材。技能本体由纯 Markdown 加少量 JSX/JS 脚手架组成，无构建、无运行时：SKILL.md 作为入口编排整个流程，system-prompt.md 沉淀设计方法论与工艺标准，references/ 中按 Cursor、Claude Code、Codex 分别维护工具映射，built-in-skills/ 提供幻灯片、移动端、导入导出等专项 Prompt，starter-components/ 预置 iOS/Android/macOS/浏览器外壳、画布、幻灯片舞台、时间轴动画引擎、可调参面板等组件。功能矩阵覆盖高保真设计、原型、线框、设计系统、Figma .fig 离线解码、GitHub 仓库导入、PDF/PPTX/Canva 导出、Gemini 图像生成等。推荐搭配 Claude Opus 4.8，并通过 npx skills add 一键安装到 70+ 种 Agent，或直接把仓库 URL 丢给 Agent 自行拉取。
在 Agent Skill 这一新兴形态尚未形成事实标准的当下，baoyu-design 提供了一个高完成度、可直接套用的范例：它把一个原本依赖云端的复杂设计工作流拆解为方法论、工具映射、专项技能、起步组件四层结构，既证明了纯 Markdown + 少量脚手架就能驱动出接近官网水准的设计产物，也展示了如何写出能跨 Cursor/Claude Code/Codex 多 Agent 复用的 Skill。对于想理解 Skills 生态、构建自有 Agent 能力包，或希望把设计环节彻底纳入代码仓库工作流的团队，这是一份兼具工程示范与即用价值的参考实现。

Agentic SOP Kit ⭐ 159

把人工SOP变成确定性闸门工作流

Python 3.8+Claude Code PluginJSON SchemaMermaidShell Hooks
agentic-sop-kit 是一个 Claude Code 插件，旨在把人工执行的标准作业流程（Human SOP）工程化为 LLM 可以安全、可重复执行的 agentic workflow。它定位为方法论加可携工具包，而不是聊天机器人，专为受监管、高风险、不能出错的场景设计。项目通过四个机制系统性封堵 LLM 的常见失败模式：事实仅来源于输入、缺失内容必须标注【待補】以杜绝臆造；确定性逻辑全部写进代码、硬闸门 hermetic 且零 LLM 介入以避免假自主；所有产出标记为 DRAFT 并由人工核准以保证可追责；通过 audit skill 与 Stop-hook 回归闸门防止退化为 mega-agent。核心引擎 run.py 提供 cmd_gate、schema_gate、trace_gate、recompute_gate 四种逐步硬闸门，支持 cmd 白名单、--plan 乾跑、forward-only branch、map_over 逐项处理，以及封顶自动修复 fix-loop（默认 3 次）。插件包含两个自动触发 Skill（agentic-sop 与 agentic-workflow-audit）、一个 /sop-flow 指令以及 SessionStart 与 Stop 两个 Hook。kit/ 目录可复制到任意专案，附 FE、BE、DB、AI 免依赖范例。
在 agentic 工程普遍滑向 mega-agent 与 prompt 堆叠的当下，这个项目用一种克制的工程哲学回应 LLM 的可信度问题：把可确定的事交还给代码，把不可确定的事标注为待补，把高风险判定交还给人。它把『硬闸门、回归测试、人工核准』三件事制度化为插件级强制力，而非散文式承诺，对受监管行业落地 AI 工作流具有方法论参考价值。
