错误发现技能 (Error Discovery Skill) ⭐ 76

为 AI 智能体提供交互式错误分析能力

Python 标准库 HTTP ServerHTML/CSS/JavaScript (单文件)Markdown (SKILL.md)JSONL/CSV/JSON 数据格式聚类算法
Error Discovery Skill 是一个面向 AI 智能体的交互式错误分析技能包，核心定位是帮助开发者对 LLM 输出或 Agent 轨迹数据集进行系统化的失败模式挖掘。用户只需提供一份 JSONL、CSV 或 JSON 格式的数据文件，智能体便会自动识别内容类型（文章、Agent 轨迹、代码或结构化输出等），并基于 Gestalt 视觉设计原则（颜色编码类别、间距体现层级、透明度表达重要性）生成可视化编码方案。随后它会通过 Python 标准库启动本地服务，渲染一个零依赖的单文件 HTML 审阅界面。系统会对数据进行聚类，挑选聚类代表与随机样本作为初始审阅集，并进入一个交互循环：实时监控标注者的自由文本批注，自动归类失败模式，跟踪覆盖度，并主动推荐新样本以填补空白区域。整个技能以纯 Markdown 形式承载于 SKILL.md，可被 Claude Code 等任意能读取文件的 Agent 直接加载使用。

OpenTag ⭐ 330

Slack 与 GitHub 的 @agent 智能路由器

Slack Events APIGitHub Webhooks / AppsOpenAI CodexAnthropic Claude CodeWebhook 路由服务
OpenTag 是一个开源的 @agent 提及（mentions）路由系统，旨在为 Slack 与 GitHub 两大协作平台提供统一的 AI 编码助手入口。它的核心定位是：当团队成员在 Slack 频道或 GitHub Issue、PR 评论中通过 @agent 形式标记任务时，OpenTag 会自动捕获这些被标记的请求，将其路由到合适的 AI 编码代理（如 OpenAI Codex、Anthropic Claude Code 等），并把执行结果回写到原始的讨论线程中，形成完整的对话闭环。 这种设计把原本割裂的「沟通工具」与「AI 编码工具」整合到团队日常工作流中，开发者无需切换上下文，只需在熟悉的 Slack 或 GitHub 评论框里 @ 一下，就能让 AI 完成代码生成、Bug 修复、PR 审查、问题分析等任务，结果直接以线程回复形式呈现，便于团队协作讨论与追溯。 项目采用开源策略，意味着团队可以自部署、自定义路由规则，避免将代码与对话内容交给闭源 SaaS。从功能定位看，它解决了多 AI Agent 并存时的「调度与分发」问题，可视为面向团队协作场景的 AI Agent 网关。由于 README 内容极简，具体技术栈尚未公开披露，但从功能推断需要 Slack Events API、GitHub Webhooks、以及 Codex、Claude Code 的 CLI 或 API 集成能力。
随着 Codex、Claude Code 等命令行 AI 编码助手快速普及，团队面临的新问题是「如何把这些单机 CLI 工具接入协作流程」。OpenTag 提出了一个极简却务实的答案：用 @mention 作为通用调用协议，把 Slack 和 GitHub 变成 AI Agent 的前端界面。它代表了一类新兴的「AI Agent 网关」产品形态，对希望在团队中规模化使用多种 AI 编码工具、又不愿被单一厂商锁定的工程组织而言，具有较强的参考与落地价值。
