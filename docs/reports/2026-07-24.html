Agent Council 是一款面向 Claude Code 的多智能体协作插件（Skill），灵感来自 Karpathy 的 LLM Council。它通过调用用户本地已安装的多个 AI CLI 工具（如 Claude Code、Codex CLI、Gemini CLI 等）来收集不同模型的独立意见，再由一个可配置的 Chairman（主席）智能体进行综合汇总，输出最终建议。与直接调用各厂商 API 的方案不同，Agent Council 复用现有 CLI 订阅，无需额外 API 费用。它采用三阶段流程：并行分发问题、收集展示各方观点、由主席合成结论，支持通过 npx 一键安装或作为 Claude Code 插件市场包安装，并可通过 council.config.yaml 灵活配置成员、命令、表情与颜色。脚本支持前台一次性执行与后台作业模式（start/status/results/clean），可与宿主 CLI 的原生 Plan/Todo UI 联动，适合需要多视角决策的开发场景。

该项目是一个基于 MIT 协议开源的免费 AI 网关服务，主打「Never stop coding（编码不中断）」的理念。它通过单一统一的 API 端点，聚合了 290+ 家 AI 服务提供商（其中 90+ 家提供免费额度），覆盖 500+ 个主流大模型，包括 Kimi、Claude、GPT、OpenAI、Gemini、GLM、DeepSeek、MiniMax 等。项目在生态兼容性上做得非常全面，可无缝对接 Claude Code、Codex、Cursor、OpenCode、Cline、Copilot 等主流 AI 编程工具，开发者无需修改现有工作流即可切换后端模型。核心能力方面，项目实现了配额感知的自动回退（Quota-aware auto-fallback）机制，当某个服务商额度耗尽或不可用时会自动切换到备用通道，保障调用稳定性；同时采用 RTK + Caveman 压缩算法，能够为 prompt 和上下文节省 15%–95% 的 token 消耗，显著降低使用成本。协议层面支持 MCP（Model Context Protocol）与 A2A（Agent-to-Agent）通信，并提供桌面客户端和 PWA 两种交付形态，兼顾本地体验与跨平台便捷性。项目由 500+ 位贡献者共同维护，社区规模可观。
在 AI 编程工具百花齐放但 API 成本高企的当下，这个项目提供了一条低门槛甚至零成本的路径：开发者用一个端点就能横跨 290+ 服务商，配合自动回退和 token 压缩，实际能把成本压到极低。加上对 Claude Code、Cursor 等主流工具的原生兼容，迁移几乎零成本；MIT 协议 + 500+ 贡献者的社区规模也说明它不是玩具级项目，对追求模型多样性与成本控制的团队非常值得关注。

Karpathy 风格 Claude Code 行为准则 ⭐ 169

把 Karpathy 踩坑心得变成 Claude Code 规则

Claude CodeCLAUDE.md 约定Cursor Project Rules (.mdc)Markdown
本项目是一份面向 Claude Code 的行为准则集合，灵感源自 Andrej Karpathy 关于大语言模型在编写代码时常见失误的公开笔记。作者将 Karpathy 指出的典型问题——擅自假设、隐藏困惑、过度设计、越界修改无关代码等——提炼为四条可执行的原则，并以一份 CLAUDE.md 文件的形式落地，使 AI 编程助手在真实项目中能够稳定遵循。四条原则分别是：Think Before Coding（先思考再动手，显式陈述假设、主动提出多种解释、必要时反问澄清）、Simplicity First（追求最小可行实现，拒绝投机性抽象与冗余错误处理）、Surgical Changes（外科手术式修改，只触碰必要行，不擅自改动风格与相邻代码）、Goal-Driven Execution（以可验证的成功标准替代模糊指令，例如先写复现测试再修复 Bug）。项目提供两种安装方式：通过 fastsetup 以 Claude Code 插件形式全局启用，或将原则追加到项目根目录的 CLAUDE.md。仓库还内置了 .cursor/rules/karpathy-guidelines.mdc，使 Cursor 用户也能复用同一套规则。整体偏向「谨慎优先于速度」，适合非平凡工程任务。
在「AI 编程助手」泛滥的当下，多数项目关注如何让模型写更多代码，而这份准则反其道行之，聚焦如何让模型少犯错、少越界、少过度设计。它把顶尖研究者的直觉经验固化为工程可复用的配置文件，是 Prompt 工程从「技巧」走向「规范」的一个典型样本，值得任何使用 Claude Code 或 Cursor 的团队直接借鉴。
