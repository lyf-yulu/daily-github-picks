给AI Agent上下文瘦身的本地代理

Node.js (>=20)TypeScriptSQLiteSSE 流式代理Claude Code Plugin
tokdiet 是一个本地流式反向代理，部署在 AI 编程代理（Claude Code、Cursor、Codex）与模型 API（Anthropic、OpenAI、Gemini、MiniMax）之间。它对每一次请求计量 token 数量与美元成本，对臃肿的上下文进行智能压缩以削减按 token 计费的 API 开销，同时通过 shadow-eval 影子评测验证回答质量未下降。项目核心理念是把上下文视作虚拟内存：热数据保留，冷数据可恢复地分页到本地 SQLite，而不是盲目删除。它对 Claude Code 的 prompt caching 和 extended thinking 特性做了专门兼容，遇到 cache_control 边界与签名 thinking 块绝不改写，并采用 fail-open 策略确保异常时透明转发。基于 66 任务 A/B 基准测试，token 输入降低约 71%，质量与基线持平（64/66 vs 63/66）。配套提供 ccusage 风格的本地仪表盘、SQLite 存储和 Claude Code 插件市场安装方式，API key 全程不落盘不入日志。
在一众只会展示账单或盲目压缩的工具中，tokdiet 是少数同时做到削减成本并以 LLM-judge + 影子评测量化证明质量未退化的方案，并对 Claude Code 的缓存与 thinking 块做了精细兼容。
