bazi-ziwei-skill 是一个遵循 SKILL.md 开放标准的命理分析 Skill，可装入 Claude Code、Claude Desktop、Codex、Cursor、Hermes、OpenClaw 等支持该标准的 AI Agent，专注解决纯 LLM 排盘容易把日柱、日主、格局算错的问题。项目通过算法层完成八字四柱、紫微十二宫、大运流年的精准排盘，再补一层格局、旺衰、调候、刑冲合害、盖头截脚的算法解读，最后把结构化文本喂给 LLM 做有依据的分析。它提供三种分析模式（八字独立、紫微独立、八字+紫微综合印证）与两种呈现形态（Markdown 长文深度版、单文件 HTML 水墨风海报版），其中综合印证模式会把两套独立体系的结论做交叉对账，校验主轴是否一致、人生窗口是否对齐、冲突时听谁，是项目区别于普通 LLM 加排盘工具的核心增量。排盘核心算法 vendored 自开源项目 Yiqi（MIT），农历换算依赖 lunar-typescript，所有计算在本地完成、无需联网，运行产物默认 gitignore，隐私友好。仓库还附带命令行脚本支持脱离 Agent 直接生成 JSON、文本盘与 HTML 海报。
命理类 AI 应用普遍存在 LLM 自行排盘导致基础数据错误的问题，本项目用算法层兜底排盘、用补层算法降低 LLM 幻觉、再用八字与紫微的交叉印证给出可信度更高的结论，工程化思路清晰；同时它是少见的同时兼容多个主流 Agent 的中文垂类 Skill 范例，对想做 SKILL.md 生态或垂直领域 AI Agent 的开发者有直接参考价值。

architect-loop 是一个面向 Claude Code 的技能（skill）集合，旨在把 Claude Fable 与 OpenAI Codex CLI 串成一条以仓库为中心的跨厂商智能体协作循环：Fable 负责规划、规约（spec）、验收门禁（gates）的撰写以及最终评审与合并，GPT-5.5 Codex 负责在隔离的 git worktree 中并行实现与研究，仓库本身则承担长期记忆。项目提供两条主命令：/architect 驱动构建循环，先由 Fable 拆出 1–4 条互不冲突的车道并把验收门禁写入 docs/gates/，再分发新鲜上下文的 codex exec（xhigh）作为隔离构建者，构建者无提交权限，Fable 亲自跑门禁命令并对照 spec 审 diff 后才合并；/architect-research 驱动研究循环，先用廉价 Codex Scout 做约 10 次检索的主题测绘，再由 Fable 设计 3–6 条按主题定制的车道，并行执行带有搜索预算、饱和停止、URL+日期+引文+置信度规范的研究员，最后由 Fable 做对抗性证伪并撰写带引用的决策报告。所有设计选择均在 DESIGN.md 中给出文献引用，强调弱规划器危害大于弱执行器、可见测试会被刷分、记忆文件易腐化等失败模式，并通过冻结外部门禁、worktree 隔离、HANDOFF 短目录等机制规避。仅依赖用户已有的 Claude 与 ChatGPT 订阅，无需 API key。
它把当下热门的「多智能体协作」从口号落到了可复现的工程模式：明确分工（强规划器+强执行器跨厂商组合）、强制外部门禁、worktree 物理隔离、仓库即记忆，并且每条规则都给出研究文献支撑而非凭直觉设计。对于想要在真实代码库里跑 agent loop 又担心幻觉、串扰与不可审计的人来说，是一份难得的、零 API key 即可上手的参考实现。

MetaHarness 智能体外壳生成器 ⭐ 198

为任意仓库一键生成专属 AI 代理外壳

Node.jsTypeScriptnpm/npx CLIModel Context Protocol (MCP)GitHub Actions见证签名 / 溯源验证Mermaid (文档)RVM 微管理程序沙箱
MetaHarness（agent-harness-generator）是一个面向 AI 智能体的「元外壳」生成器，定位是『生产代理框架的工厂』而非又一个代理框架。开发者通过 `npx metaharness` 或浏览器 Studio，可在 60 秒内基于 GitHub 仓库 URL 或空白模板，铸造出一个仓库感知的定制化代理外壳：自带独立品牌的 npx CLI、本地 MCP 服务器、项目级记忆命名空间、由真实文件结构生成的技能与斜杠命令、治理策略，以及由见证签名（witness-signed）保障的发布溯源与门禁。生成产物是可发布到 npm 的 zip 包，支持 Claude Code、OpenAI Codex、pi.dev、Hermes、OpenClaw、RVM、GitHub Copilot、OpenCode 八大交互宿主以及 GitHub Actions CI/CD 共九种宿主。项目还提供三大特色子能力：`metaharness score` 在不执行代码的前提下评估仓库适配度、构建概率、工具安全性与每次运行成本；`@metaharness/router` 根据历史评测日志将每个请求路由到『刚好够用』的最便宜模型；`@metaharness/darwin` 的 Darwin 模式让外壳在沙箱中自我变异并保留可度量的改进，模型冻结而外壳进化。MCP 采用默认拒绝、可分级（off/local/remote）的适配层设计，确保安全可审计。
在 AI 代理工具同质化严重的当下，MetaHarness 反向押注『模型可替换、外壳才是产品』，把通常被框架内嵌的 CLI、MCP、记忆、治理、签名发布等能力下沉为可铸造、可裁剪、可自我进化的工件，并用 score+router+Darwin 把成本工程做到了一等公民的位置。九宿主统一输出与默认拒绝的安全模型也让它在企业内部代理标准化场景具备实用价值。

tokdiet 令牌瘦身代理 ⭐ 68

给 AI Agent 省钱的本地代理

Node.js (>=20)TypeScriptSSE 流式代理SQLiteMIT LicenseClaude Code Plugin
tokdiet 是一个本地流式反向代理，部署在 AI 编码助手（Claude Code、Cursor、Codex）与模型 API（Anthropic、OpenAI、Gemini、MiniMax）之间。它对每一次请求计量 token 和美元成本，并对臃肿的上下文进行智能压缩，从而显著降低按量计费的 API 开销。项目最大的差异点是用真实的 66 任务 A/B 基准（198 次配对运行）证明压缩后质量未下降：在 MiniMax-M3 上输入 token 减少 71%，质量与基线持平（63/66 vs 64/66）。它将上下文当作虚拟内存处理，冷数据可恢复地分页到本地 SQLite 而非盲删，并对 Claude Code 的 prompt 缓存与 extended thinking 块做了兼容保护。代理采用 SSE 增量转发不破坏流式体验，API key 仅透传不落库不入日志，遇错自动 fail-open 回退透明转发。配套提供 ccusage 风格的本地仪表盘、预算限制以及 shadow-eval 质量守护。
市面上的上下文优化工具大多盲删 token 而无法证明质量未受损，tokdiet 通过可复现的 198 次配对基准 + LLM-judge 给出量化证据，并在工程上同时解决了 prompt 缓存失效、thinking 块签名校验等真实 Agent 场景下的隐藏陷阱。
