OpenCodex ⭐ 102

让 Codex 接入任意 LLM 的本地代理

BunTypeScriptOpenAI Responses APIAnthropic Messages APIGoogle Gemini APIlaunchd / systemd / Task Scheduler
OpenCodex 是一个轻量级本地代理工具，专为 OpenAI Codex CLI、App 和 SDK 设计，核心目标是打破 Codex 只能使用 OpenAI 模型的限制，让开发者可以将 Claude、Gemini、Grok、DeepSeek、Kimi、Qwen、Ollama 等 40 余个主流 LLM 提供商无缝接入 Codex 工作流。 其工作原理是在本地启动一个监听 10100 端口的代理服务，将 Codex 发出的 OpenAI Responses API 请求拦截并翻译成目标提供商所支持的协议格式，目前实现了五种协议适配器：Anthropic Messages、Google Gemini、Azure OpenAI、OpenAI Responses 直通，以及覆盖绝大多数兼容端点的 OpenAI Chat Completions。流式输出、工具调用、推理 token、图像理解等能力在双向传输中均得到完整支持。 在使用体验上，OpenCodex 提供 Web 仪表盘（`ocx gui`）和交互式 CLI（`ocx init`）两种配置方式，支持通过 provider/model 语法精确路由模型，路由后的模型会直接出现在 Codex App 的模型选择器中。OAuth 登录支持 Anthropic、xAI 和 Kimi，避免手动管理 API Key。代理可作为系统服务（launchd / systemd / Task Scheduler）在后台常驻，`ocx stop` 可完整还原 Codex 原始配置，无任何残留。 技术上基于 Bun 运行时构建，支持 macOS、Linux、Windows 三平台原生运行，无需 WSL。整体设计极为克制，安装一条命令，启动一条命令，对现有 Codex 使用习惯零侵入。
Codex CLI 生态正快速扩张，但官方仅支持 OpenAI 模型，OpenCodex 以极低的接入成本（一条 npm install）填补了这一空白。其协议适配层设计通用，理论上可覆盖任何 OpenAI 兼容端点，对于希望在企业内网用私有部署模型（Ollama/vLLM）驱动 Codex 的团队而言，这是目前最轻量、侵入性最低的方案。
